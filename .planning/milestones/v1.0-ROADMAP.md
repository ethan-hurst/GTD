# Milestone v1.0: MVP

**Status:** SHIPPED 2026-01-31
**Phases:** 1-8
**Total Plans:** 35

## Overview

This roadmap delivers a complete web-based GTD (Getting Things Done) productivity system in 8 phases. The journey follows the natural GTD methodology flow: establish offline-first storage foundation, build capture and processing workflow, add core lists (next actions, projects, waiting-for, someday/maybe), implement weekly review (the critical success factor), guide new users through GTD concepts, and finally integrate calendar viewing. Each phase delivers a complete, verifiable capability that builds toward the core value: nothing falls through the cracks.

## Milestone Summary

**Key Decisions:**
- SvelteKit 2 + Svelte 5 $state runes for modern reactive patterns
- Dexie 4.x with EntityTable for type-safe IndexedDB operations
- Tailwind v4 with Vite plugin (no PostCSS config needed)
- Schema extension over separate tables for waiting/someday fields
- @event-calendar/core over FullCalendar for smaller bundle and native Svelte 5 support
- Settings table for key-value persistence (review dates, onboarding state, feature tracking)
- Single-key keyboard shortcuts (n/p/w/s/r/c//) extending progressively across phases
- ProcessingFlow as central GTD routing hub (inbox → all destination lists)

**Issues Resolved:**
- Tailwind v4 dark mode incompatibility (required @custom-variant dark)
- Dexie id:0 constraint error on second insert (removed explicit id)
- Search index corruption (added fallback full-scan)
- Svelte 5 proxy objects breaking svelte-dnd-action ($state.snapshot() fix)
- @event-calendar/core named export vs default import

**Issues Deferred:**
- Two-way Microsoft Outlook/Teams calendar sync (deferred to v2)
- Natural language parsing for quick capture (deferred to v2)
- Sequential vs parallel project types (deferred to v2)
- Custom perspectives / saved filter combinations (deferred to v2)

**Technical Debt Incurred:**
- None (tech debt cleanup completed as quick-001 before milestone completion)
