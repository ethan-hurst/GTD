Update `src/lib/components/ProcessingFlow.svelte` to add a "time-specific" option in the GTD processing workflow that routes items to the calendar.

Per CONTEXT.md: "Processing flow gains new option: 'Is this time-specific? -> Add to calendar' alongside existing defer/delegate paths."

The current processing flow has these steps:
- Step 1 (actionable): "Is this actionable?" -> Yes/No
- Step 2 (two-minute): "Can you do this in 2 minutes?" -> Yes (do it now) / No (continue)
- Step 3 (delegate-or-defer): "Delegate or defer?" -> Delegate / Defer (next action)

Add a new option in the `delegate-or-defer` step (or create a new step between two-minute and delegate-or-defer):

**Approach**: Add a "Schedule it" button alongside "Delegate" and "Defer" in the delegate-or-defer step. This represents the GTD question "Is this time-specific?"

When user clicks "Schedule it":
1. Navigate to a new step: `'schedule-event'`
2. Show a simplified event creation form inline (matching the ProcessingFlow's inline expansion pattern):
   - Title: pre-filled from the inbox item's title
   - Date/time: datetime-local input for start time
   - End time: datetime-local input (defaults to start + 1 hour)
   - Save button: creates event via `addEvent()` from operations.ts, then removes the inbox item (marks as processed), calls onProcessed()
3. The inbox item is consumed — it becomes a calendar event instead of a next action

Implementation details:
- Add `'schedule-event'` to the Step type union
- Import `addEvent` from `$lib/db/operations`
- Add `schedule-event` to stepIndicators
- Add navigation: from delegate-or-defer, "Schedule it" -> schedule-event step
- Add back navigation: from schedule-event, back -> delegate-or-defer
- In schedule-event step UI:
  - Show item title (read-only or editable)
  - Start datetime input
  - End datetime input
  - "Add to Calendar" primary button
- On save: add event to database, delete/update the inbox item as processed, call onProcessed(), show toast "Added to calendar"

Update step indicator text for delegate-or-defer to something like "Step 3 of 3" (unchanged, since schedule-event is a branch).

The button order in delegate-or-defer should be: "Defer (Next Action)" | "Delegate" | "Schedule it"
This presents the most common action first (defer), then delegate, then schedule.
