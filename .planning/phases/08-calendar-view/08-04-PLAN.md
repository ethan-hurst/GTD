  <files>src/lib/components/EventForm.svelte</files>
  <action>
Create `src/lib/components/EventForm.svelte` — a slide-in panel (matching existing detail panel patterns like ActionDetailPanel) for creating and editing calendar events.

Props:
```typescript
interface EventFormProps {
  event?: CalendarEvent;        // Existing event for edit mode, undefined for create
  initialDate?: Date;           // Pre-fill start time when clicking a time slot
  onSave: () => void;          // Callback after save
  onClose: () => void;         // Close the form
}
```

Form fields (all using native HTML inputs, matching existing app patterns):
1. **Title**: text input, required, autofocus on open
2. **Start date/time**: `<input type="datetime-local">` — pre-filled from initialDate or event.startTime
3. **End date/time**: `<input type="datetime-local">` — defaults to start + 1 hour
4. **All day**: checkbox — when checked, hide time portions and use date-only inputs
5. **Location**: text input, optional
6. **Notes**: textarea, optional, 3 rows
7. **Color**: A row of 6-8 color swatches (preset colors: blue, green, red, purple, orange, teal, pink, gray). Click to select. Show selected with a ring/border. Default: blue.
8. **Project link**: dropdown select of active projects (load from getAllProjects). Optional, value is projectId. Include "(None)" option.
9. **Recurrence**: dropdown select with options: "Does not repeat", "Daily", "Weekly", "Monthly". When selected, store as RRULE string:
   - Daily: "FREQ=DAILY"
   - Weekly: "FREQ=WEEKLY"
   - Monthly: "FREQ=MONTHLY"
   - Does not repeat: undefined/null

Buttons:
- **Save**: primary button (blue), calls calendarState.addEvent (create) or calendarState.updateEvent (edit)
- **Delete**: red text button, only shown in edit mode, calls calendarState.deleteEvent with confirmation
- **Cancel**: secondary button, calls onClose

Layout: Right-side panel that slides in (use `transition:slide` from svelte/transition, same pattern as ActionDetailPanel). Width: 350-400px. The panel overlays/pushes the calendar content.

Date/time handling: Convert between datetime-local input format (string "YYYY-MM-DDTHH:MM") and Date objects. Use helper functions within the component for conversion.

When user saves:
- Create mode: call `calendarState.addEvent({ title, startTime, endTime, allDay, location, notes, color, projectId, source: 'manual', rrule })` then onSave()
- Edit mode: call `calendarState.updateEvent(event.id, { ...changes })` then onSave()
- Delete: call `calendarState.deleteEvent(event.id)` then onClose(), show toast "Event deleted"

Use `svelte-5-french-toast` for save/delete notifications (matching existing patterns).
