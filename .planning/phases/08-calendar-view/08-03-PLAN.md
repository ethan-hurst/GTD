<task type="auto">
  <name>Task 1: Create EventCalendar wrapper component</name>
  <files>src/lib/components/EventCalendar.svelte</files>
  <action>
Create `src/lib/components/EventCalendar.svelte` that wraps @event-calendar/core for Svelte 5.

The component should:

1. Import the calendar library and plugins:
   ```typescript
   import Calendar from '@event-calendar/core';
   import TimeGrid from '@event-calendar/time-grid';
   import DayGrid from '@event-calendar/day-grid';
   import Interaction from '@event-calendar/interaction';
   import List from '@event-calendar/list';
   ```

2. Accept props:
   ```typescript
   interface EventCalendarProps {
     events: CalendarEvent[];
     currentView: 'dayGridMonth' | 'timeGridWeek' | 'timeGridDay';
     currentDate: Date;
     onEventClick?: (event: CalendarEvent) => void;
     onDateClick?: (date: Date) => void;
     onEventDrop?: (event: CalendarEvent, newStart: Date, newEnd: Date) => void;
     onEventResize?: (event: CalendarEvent, newStart: Date, newEnd: Date) => void;
   }
   ```

3. Map CalendarEvent[] to the library's event format. The @event-calendar/core expects events in the format:
   ```typescript
   {
     id: string | number,
     title: string,
     start: Date,
     end: Date,
     allDay?: boolean,
     backgroundColor?: string,
     extendedProps?: Record<string, any>
   }
   ```
   Use `$derived` to reactively map `events` prop to this format. Store extra fields (location, notes, projectId, source, rrule) in `extendedProps`.

4. Configure the calendar options:
   - `view`: bound to currentView prop
   - `date`: bound to currentDate prop
   - `plugins`: [TimeGrid, DayGrid, Interaction, List]
   - `headerToolbar`: { start: '', center: '', end: '' } (we handle our own toolbar in the page)
   - `editable`: true (enables drag-and-drop)
   - `eventClick`: callback that extracts CalendarEvent from extendedProps and calls onEventClick
   - `dateClick`: callback that calls onDateClick with the clicked date
   - `eventDrop`: callback that calls onEventDrop with updated times
   - `eventResize`: callback that calls onEventResize with updated times
   - `allDaySlot`: true
   - `slotMinTime`: '06:00:00'
   - `slotMaxTime`: '22:00:00'
   - `nowIndicator`: true (show current time line)
   - `dayMaxEvents`: true (collapse in month view)
   - `height`: '100%'

5. Mount the calendar into a container div. The @event-calendar/core library uses `new Calendar({ target, props })` Svelte component pattern. Since this is a Svelte component wrapping another Svelte component, use `<Calendar {plugins} {options} />` syntax if supported, or mount imperatively via `onMount` with a target element.

6. Style the calendar container to fill available space. Import the library's CSS:
   ```
   import '@event-calendar/core/index.css';
   ```
   Add dark mode overrides using Tailwind's dark: classes or CSS custom properties on the wrapper div to ensure the calendar respects the app's dark mode.

Important: The @event-calendar library is a Svelte component itself. Check its actual Svelte integration API — it may export a component directly usable as `<Calendar />` rather than requiring imperative mounting. Read `node_modules/@event-calendar/core` to verify the export pattern after Plan 01 installs it.
  </action>
  <verify>Run `npm run check` to verify TypeScript compilation. Run `npm run dev` and navigate to /calendar (once page is created in Task 2) to see the calendar renders.</verify>
  <done>EventCalendar.svelte wraps @event-calendar/core with reactive props. Events are mapped from CalendarEvent format to library format via $derived. Callbacks for click, drop, and resize are wired. Calendar respects dark mode.</done>
</task>
