## Decisions Made

**Mobile navigation pattern:** Chose hamburger menu with slide-out drawer over bottom tab bar because:
- Accommodates 8 main navigation items without crowding
- Matches desktop sidebar structure for consistency
- Familiar pattern users expect on mobile

**FAB placement:** Bottom-right corner with safe area insets for:
- Quick access to most common action (inbox capture)
- Right-handed and left-handed thumb reach
- Doesn't block content (56px diameter vs full-width button)

**Conditional rendering approach:** Separate mobile/desktop layout blocks instead of CSS-only responsive because:
- Complete control over mobile component tree (no desktop components mounted)
- Simpler to reason about than complex CSS media queries
- Easier to add mobile-specific features later

**Drawer close triggers:** Close on Escape, backdrop click, AND nav link click because:
- Nav link click: User achieved their goal (navigate), drawer should dismiss
- Backdrop click: Clear "cancel" affordance
- Escape key: Keyboard accessibility
