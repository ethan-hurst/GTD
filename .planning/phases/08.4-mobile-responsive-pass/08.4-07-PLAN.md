  <action>
Using svelte-gestures `pan` action, add swipe-to-complete (right) and swipe-to-delete (left) to list items. Only enable swipe gestures on touch devices (when mobileState.isMobile is true).

**Pattern to apply to each item component:**

1. Import: `import { pan } from 'svelte-gestures';` and `import { mobileState } from '$lib/stores/mobile.svelte';`

2. Add state for swipe tracking:
```typescript
let swipeOffset = $state(0);
let isRevealing = $state(false);
const SWIPE_THRESHOLD = 80; // pixels before action triggers
```

3. Handlers:
```typescript
function handlePan(event: CustomEvent) {
  swipeOffset = Math.max(-120, Math.min(120, event.detail.x));
  isRevealing = Math.abs(swipeOffset) > 20;
}

function handlePanEnd(event: CustomEvent) {
  if (event.detail.x > SWIPE_THRESHOLD) {
    // Swipe right: complete item
    completeItem(); // or appropriate action per component
  } else if (event.detail.x < -SWIPE_THRESHOLD) {
    // Swipe left: delete item
    deleteItem(); // or appropriate action per component
  }
  swipeOffset = 0;
  isRevealing = false;
}
```

4. Template structure (wrap existing item content):
```svelte
<div class="relative overflow-hidden">
  <!-- Revealed action backgrounds -->
  {#if isRevealing && mobileState.isMobile}
    <div class="absolute inset-y-0 left-0 w-full flex items-center px-4 bg-green-500 text-white">
      <svg class="w-6 h-6"><!-- Checkmark icon --></svg>
      <span class="ml-2 text-sm font-medium">Complete</span>
    </div>
    <div class="absolute inset-y-0 right-0 w-full flex items-center justify-end px-4 bg-red-500 text-white">
      <span class="mr-2 text-sm font-medium">Delete</span>
      <svg class="w-6 h-6"><!-- Trash icon --></svg>
    </div>
  {/if}

  <!-- Main item content (slides) -->
  <div
    use:pan={{ delay: 0 }}
    onpan={handlePan}
    onpanend={handlePanEnd}
    style="transform: translateX({mobileState.isMobile ? swipeOffset : 0}px); transition: {isRevealing ? 'none' : 'transform 0.2s ease-out'};"
    class="relative z-10 bg-white dark:bg-gray-950">
    <!-- Existing item content here -->
  </div>
</div>
```

Note: Only apply `use:pan` and transform when on mobile to avoid breaking desktop hover/click behavior. Use `{#if mobileState.isMobile}` for conditional gesture binding, or use `style:transform` that only applies offset when mobile.

IMPORTANT: svelte-gestures uses `use:pan` action directive with event names like `onpan` and `onpanend`. Check svelte-gestures docs for exact Svelte 5 event syntax. The library may use `on:pan` (Svelte 4) or a different pattern for Svelte 5. If it uses Svelte 4 `on:pan` syntax, that still works in Svelte 5 compatibility mode.

**Per-component actions:**
- **InboxList items**: Swipe right = start processing that item, Swipe left = delete
- **ActionItem**: Swipe right = mark complete, Swipe left = delete
- **WaitingForItem**: Swipe right = mark received/complete, Swipe left = delete
- **SomedayMaybeItem**: Swipe right = promote to action/project (or just mark active), Swipe left = delete

Also configure existing **svelte-dnd-action** for touch:
- Where `use:dndzone` is used, add `flipDurationMs` and check that `dragDisabled` is not set on mobile
- svelte-dnd-action supports touch natively with `type: "touch"` handling
- Add `morphDisabled={false}` if needed for smooth animations on touch

Add swipe left/right gesture to the calendar view container for day navigation on mobile.

1. Import: `import { swipe } from 'svelte-gestures';`

2. Handler:
```typescript
function handleCalendarSwipe(event: CustomEvent) {
  // Only navigate when in Day view on mobile
  if (!mobileState.isMobile || calendarState.currentView !== 'timeGridDay') return;

  if (event.detail.direction === 'left') {
    goForward(); // Next day
  } else if (event.detail.direction === 'right') {
    goBack(); // Previous day
  }
}
```

3. Apply to the calendar container div:
```svelte
<div
  use:swipe={{ timeframe: 300, minSwipeDistance: 60 }}
  onswipe={handleCalendarSwipe}
  class="flex-1 overflow-hidden relative touch-pan-y">
  <!-- EventCalendar component -->
</div>
```

Key details:
- `touch-pan-y` CSS class allows vertical scrolling while capturing horizontal swipes
- Only active on mobile AND in Day view (Week/Month views need native scroll)
- Swipe threshold: 60px minimum distance, 300ms timeframe
- Ensure this doesn't conflict with event-calendar's own touch handling (drag events, tap to create)
- The swipe gesture is on the outer container, not on individual calendar events
- Test that vertical scrolling (through the day's hours) still works

IMPORTANT: Check svelte-gestures API for Svelte 5 compatibility. The `onswipe` event name may differ. Check if it's `on:swipe` or `onswipe` in the latest version.
