## Decisions Made

**Custom breakpoint values:**
- phone: 360px (minimum supported mobile)
- phablet: 640px (large phones)
- tablet: 768px (tablets)
- desktop: 1024px (desktop)

**Mobile detection approach:**
- JS-level detection (mobileState store) complements CSS breakpoints (Tailwind)
- Use mobileState when logic needs to know viewport size (e.g., auto-collapse sidebar on mobile)
- Use Tailwind classes when styling needs to adapt (e.g., phone:text-sm)

**Accessibility preservation:**
- Did NOT add user-scalable=no or maximum-scale=1 (violates WCAG)
- Used max(1rem, 16px) on inputs to prevent iOS zoom without blocking user zoom
