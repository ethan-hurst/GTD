**Calendar page (`calendar/+page.svelte`):**

1. Import mobileState: `import { mobileState } from '$lib/stores/mobile.svelte';`

2. Default to Day view on mobile: In the script section, add an effect or onMount logic that sets `calendarState.setView('timeGridDay')` when `mobileState.isMobile` is true on initial load. Use a flag to avoid overriding user's explicit view choice:
```typescript
let initialViewSet = false;
$effect(() => {
  if (!initialViewSet && mobileState.isMobile) {
    calendarState.setView('timeGridDay');
    initialViewSet = true;
  }
});
```

3. Restructure the toolbar for mobile. Currently it's a single flex row with left group and right group. On mobile, this overflows. Restructure:
- **Mobile toolbar** (below tablet): Stack into 2 rows
  - Row 1: Back arrow, "Today" button, Forward arrow, date label (compact)
  - Row 2: View switcher (Day/Week/Month) + New Event + Actions toggle
  - Use `flex flex-col gap-2 tablet:flex-row tablet:items-center tablet:justify-between`
- **Desktop toolbar**: Keep existing layout
- Reduce padding: `px-3 py-2 tablet:px-6 tablet:py-3`
- Date label on mobile: shorter format. For Day view use "Thu, Jan 30" not "Thursday, January 30, 2026". Add a compact dateLabel derived that uses shorter format when mobile
- Hide "Import" button text on mobile, show icon only: `<span class="hidden phablet:inline">Import</span>`
- Hide "Actions" button text on mobile, show icon only: `<span class="hidden phablet:inline">Actions</span>`
- New Event button on mobile: icon only, no text. `<span class="hidden phablet:inline">New Event</span>`
- View switcher buttons: smaller padding on mobile `px-2 py-1 tablet:px-3 tablet:py-1.5`
- All toolbar buttons: ensure min-h-10 for adequate touch targets

4. Calendar + Side Panel layout:
- On mobile: Side panel and calendar should NOT show side-by-side
- Replace `<div class="flex h-full">` with responsive behavior:
  - On mobile: Only ONE of calendar or side panel visible at a time
  - The "Actions" toggle button switches between them
  - When side panel is visible on mobile, it takes full width
  - When calendar is visible on mobile, side panel is hidden
- On desktop: Keep existing side-by-side layout (flex h-full)

5. Navigation arrows: already `p-2.5` which gives ~44px target. Verify this is maintained.

**EventCalendar.svelte:**
- Add mobile-specific calendar options:
  - `eventLongPressDelay: 500` (ms before event drag starts on touch)
  - `eventDragMinDistance: 10` (px minimum before drag starts, prevents accidental drags)
  - These may need to be passed as props or configured in the component
- Ensure the calendar container uses `touch-action: pan-y` to allow vertical scrolling while preventing horizontal browser gestures
