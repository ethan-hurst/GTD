## Decisions Made

**WeeklyReviewWizard sidebar approach:**
- Chose to hide sidebar completely on mobile rather than make it collapsible
- Replaced with compact progress indicator showing "Step X of 8"
- Rationale: Sidebar takes valuable screen space, users navigate linearly through steps on mobile anyway

**OnboardingWizard full-screen:**
- Full-screen experience on mobile (no backdrop visible, no rounded corners)
- Centered modal card with backdrop on tablet+
- Rationale: First-run experience should be immersive on mobile, use all available space

**Input font sizing:**
- Explicit `font-size: 16px` on capture input to prevent iOS zoom
- Applied via inline style to override any class-based sizing
- Rationale: iOS Safari zooms page when input <16px focused, disrupting UX
