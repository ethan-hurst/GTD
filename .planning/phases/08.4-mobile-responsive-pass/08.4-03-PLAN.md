**Inbox page (`+page.svelte`):**

- Header flex row: keep as-is (already works at narrow widths)
- Process Inbox button: already full-width, ensure min-h-11 for touch target

**InboxCapture.svelte:**
- Ensure input has `text-base` class (16px, prevents iOS zoom)
- Add min-h-11 to the capture input wrapper for touch target
- Input should be full-width on all screen sizes

**InboxList.svelte:**
- Ensure list items have adequate padding: py-3 minimum for 44px touch target
- If items have action buttons (delete, process), ensure those are min-h-11 min-w-11

**ActionList.svelte:**
- Context filter bar: on mobile, make it horizontally scrollable instead of wrapping. Use `flex overflow-x-auto gap-2 pb-2 -mx-4 px-4 tablet:mx-0 tablet:px-0 tablet:flex-wrap tablet:overflow-visible` to allow scroll on mobile but wrap on desktop
- If there's a header with title + filter together on one line, stack them on mobile: `flex flex-col tablet:flex-row`
- Reduce page padding: `p-4 tablet:p-6`
- Hide context filter text labels on very small screens if needed, keeping icons

**ActionItem.svelte:**
- Ensure each item row is min-h-11 (44px touch target)
- Text truncation: use `truncate` on title text to prevent overflow
- Action buttons (complete checkbox, etc.): min-w-11 min-h-11 with p-2 padding
- If there are context badges or tags, allow them to wrap: `flex-wrap`
- On mobile, hide secondary info (project link, context) or show below the title in a second line instead of inline

**ActionDetailPanel.svelte:**
- On mobile: render as a fixed full-screen overlay (`fixed inset-0 z-40 bg-white dark:bg-gray-950`) instead of a side panel
- On desktop: keep existing side panel behavior
- Use the custom breakpoints: check viewport or use Tailwind classes
- Add a close/back button at the top of the mobile overlay
- Ensure all form inputs inside have `text-base` for iOS zoom prevention
