**WeeklyReviewWizard.svelte (221 lines):**

The wizard has 8 steps with a progress bar, step content, and next/previous navigation. It currently uses a frosted glass footer.

Key responsive changes:
1. Wizard container: reduce padding `p-4 tablet:p-6`
2. Step progress indicator:
   - If it's a horizontal step bar, on mobile either:
     a) Show compact "Step 3 of 8" text instead of individual step dots
     b) Or make step dots smaller with less gap: `gap-1 tablet:gap-2`
   - Progress percentage with `tabular-nums` (already in place) works fine on mobile
3. Step content area:
   - Full-width on mobile
   - Text: `text-base` for instructions/descriptions
   - If step has a list of items to review (inbox count, stalled projects), ensure items are readable and tappable
   - Link buttons to navigate to specific views: min-h-11 touch target
4. Navigation footer (frosted glass with backdrop-blur-sm):
   - Buttons: `w-full phablet:w-auto`, min-h-11 for touch target
   - On mobile, stack Previous and Next buttons vertically or make them full-width side by side: `flex flex-col gap-2 phablet:flex-row phablet:justify-between`
   - Add safe area bottom padding: `pb-[env(safe-area-inset-bottom,0px)]`
5. Confetti celebration (canvas-confetti): works fine on mobile, no changes needed
6. If any step has a checklist or expandable sections, ensure each item has min-h-11 touch target

**OnboardingWizard.svelte (328 lines):**

The onboarding wizard is a full-screen overlay with 5 steps that guide new users through GTD concepts and the first capture. It already renders as an overlay.

Key responsive changes:
1. The overlay modal/card: on mobile, make it full-screen instead of a centered card
   - `fixed inset-0 z-50 overflow-y-auto` on mobile
   - `fixed inset-0 z-50 flex items-center justify-center` with inner card on desktop
   - Inner card: `w-full h-full tablet:w-auto tablet:h-auto tablet:max-w-2xl tablet:rounded-xl`
2. Step content:
   - Padding: `p-4 tablet:p-8`
   - Headings: `text-xl tablet:text-2xl`
   - Body text: `text-base` (already fine for mobile readability)
3. Illustration/icon areas: if any step has a large illustration, ensure `max-w-full` and responsive sizing
4. The real inbox capture step (step where user types their first item):
   - Input must be `text-base` (16px) to prevent iOS zoom
   - Input should be full-width
   - Submit button: full-width on mobile, min-h-11
5. Navigation buttons:
   - "Skip" and "Continue"/"Next" buttons: full-width on mobile, stacked or side-by-side
   - `flex flex-col gap-2 phablet:flex-row phablet:justify-between`
   - Each button: min-h-11
6. Step dots/progress: compact for mobile
7. Add safe area padding for bottom navigation: `pb-[env(safe-area-inset-bottom,0px)]`

**StatusBar.svelte (167 lines):**
