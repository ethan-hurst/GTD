# Phase 08.4: Mobile Responsive Pass - Context

**Gathered:** 2026-01-31
**Status:** Ready for planning

<domain>
## Phase Boundary

Make every existing view usable on mobile devices — responsive layouts, touch-friendly interactions, and mobile navigation. No new features are added; this is purely adapting the existing desktop-first GTD app to work well on phones and tablets.

</domain>

<decisions>
## Implementation Decisions

### Navigation approach
- Hamburger menu with slide-out drawer on mobile (no bottom tab bar)
- Drawer shows simplified nav only — main navigation links (Inbox, Actions, Projects, Waiting For, Someday/Maybe, Calendar, Weekly Review, Settings) without the full context list, search bar, or footer details
- Contexts are accessible within their respective views instead of in the drawer
- Drawer overlay behavior: Claude's discretion (dim overlay vs push content)
- Mobile header bar: Claude's discretion (fixed vs hide-on-scroll, with hamburger + page title + optional quick-add)

### Breakpoint strategy
- Minimum supported width: 360px (common Android baseline)
- Four breakpoints: phone (360-639px), phablet (640-767px), tablet (768-1023px), desktop (>=1024px)
- Dedicated tablet layout — sidebar may be visible but narrower, two-column layouts where appropriate
- Feature degradation approach: Claude's discretion per-feature (graceful degradation vs hiding non-essential features)

### Calendar on mobile
- Default view on mobile: Day view (most readable on small screens)
- Side panel (next actions): Replaced with a toggle button that switches between calendar view and next actions list — only one visible at a time
- Month view availability: Claude's discretion based on calendar component capabilities
- Day navigation: Swipe left/right to move between days

### Touch interactions
- Drag-and-drop reordering: Long-press to pick up, then drag to reorder (standard iOS/Android pattern)
- Swipe actions on list items: Swipe right to complete/check off, swipe left to delete or defer — needs visual hints for discoverability
- Hover state replacement: Long-press for preview/tooltip (similar to 3D Touch / Haptic Touch)
- Quick capture on mobile: Floating action button (FAB) in bottom-right corner — prominent '+' button that opens inbox capture form

### Claude's Discretion
- Drawer overlay style (dim backdrop vs push content)
- Mobile header bar behavior (fixed vs auto-hide on scroll)
- Per-feature degradation decisions (graceful degradation vs hide)
- Month view on phone (available as compact dot-calendar or tablet/desktop only)
- Exact breakpoint-specific layout adjustments for each view
- Loading and empty state adaptations for mobile

</decisions>

<specifics>
## Specific Ideas

- Navigation should feel like a standard mobile productivity app (Todoist, Things 3) — hamburger drawer is familiar and expected
- FAB for quick capture is critical — inbox capture is the most frequent GTD action and needs to be always one tap away
- Swipe actions on list items like Apple Mail / Todoist for quick processing
- Long-press for both drag reorder AND preview tooltips — need to differentiate (e.g., long-press on item body = tooltip, long-press on drag zone = reorder)
- Day view with swipe navigation for calendar feels natural on mobile (like Google Calendar)

</specifics>
