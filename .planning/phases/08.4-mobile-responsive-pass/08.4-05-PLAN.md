**ProcessingFlow.svelte (487 lines - the GTD decision tree):**

The ProcessingFlow renders inline within the Inbox page when processing items. It has multiple steps with buttons for routing decisions (Is it actionable? -> Next Action / Project / Reference / Trash / Waiting For / Someday Maybe / Calendar).

Key responsive changes:
1. Container padding: reduce from any large padding to `p-3 tablet:p-6`
2. Decision tree question text: ensure `text-base` minimum on mobile for readability
3. Routing buttons (the core interaction):
   - If buttons are in a grid or horizontal layout, make them stack vertically on mobile: `flex flex-col gap-2 phablet:flex-row phablet:flex-wrap`
   - Each button: `w-full phablet:w-auto` and `min-h-11 py-3` for 44px+ touch targets
   - Add `active:scale-[0.98]` for touch feedback
4. Form inputs within processing steps (title editing, notes, context selection):
   - All inputs: `text-base` class (16px, prevents iOS zoom)
   - Select dropdowns: `text-base min-h-11`
   - Textarea: `text-base min-h-[120px]` for comfortable mobile typing
5. Step indicator/progress: if there's a step counter or breadcrumb, ensure it fits at 360px
6. Back/cancel buttons: min-h-11, full-width on mobile
7. If any step has side-by-side fields (e.g., context + project), stack them on mobile: `flex flex-col gap-3 phablet:flex-row`

**IcsImport.svelte (232 lines - modal dialog):**

Key responsive changes:
1. Modal on mobile: full-screen overlay instead of centered modal
   - Change from centered modal (`max-w-lg mx-auto`) to: `fixed inset-0 z-50 tablet:inset-auto tablet:relative tablet:max-w-lg tablet:mx-auto`
   - On mobile: full-screen with `bg-white dark:bg-gray-950 overflow-y-auto`
   - On desktop: keep existing centered modal with backdrop
2. File picker area: full-width, tall enough for easy tap (`min-h-32`)
3. Close button: min-h-11 in top-right or as header bar on mobile
4. Import/cancel buttons: `w-full tablet:w-auto`, min-h-11
5. All text inputs: `text-base`
6. If there's a drag-and-drop zone, add text "Tap to select file" alongside "Drag and drop" since drag-and-drop doesn't work on mobile
7. File input: ensure it's accessible and tappable on mobile (the native file picker works)
8. Add safe area padding: `pb-[env(safe-area-inset-bottom,0px)]` on the modal content
