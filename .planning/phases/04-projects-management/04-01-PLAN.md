  <name>Task 1: Add project operations to operations.ts</name>
  <files>src/lib/db/operations.ts</files>
  <action>
Add a new "Project Operations" section to operations.ts (after the Action Operations section) with these functions:

1. `getAllProjects()`: Returns all active projects (type='project', completedAt undefined), sorted by created date ascending. Use the same pattern as getAllNextActions but filter on type='project'.

2. `getActionsByProject(projectId: number)`: Returns all active next-actions linked to a project. Query where type='next-action', filter completedAt undefined AND projectId matches. Return as array.

3. `getStalledProjects()`: Returns active projects that have NO active next-actions. Use the 2-query pattern from RESEARCH.md:
   - Query all active projects (type='project', no completedAt)
   - Query all active next-actions with a projectId (type='next-action', no completedAt, projectId defined)
   - Build Set of projectIds from actions
   - Return projects whose id is NOT in the Set

4. `completeProject(id: number)`: Sets completedAt to new Date() and modified to new Date(). Returns an undo function (same pattern as completeAction) that clears completedAt.

5. `moveProjectToSomeday(id: number)`: Updates type to 'someday' and modified to new Date(). Returns void.

6. `addProject(title: string, notes?: string)`: Creates a new GTDItem with type='project', the given title, optional notes, and returns the new id. Use the addItem pattern but specific to projects.

Do NOT modify the schema -- GTDItem already supports type='project' and all needed fields.
  </action>

  </verify>

  <action>
Create src/lib/stores/projects.svelte.ts mirroring the ActionState class pattern from actions.svelte.ts.

The ProjectState class should have:

**State fields ($state):**
- `items`: GTDItem[] -- all active projects
- `stalledIds`: Set&lt;number&gt; -- IDs of projects with no next actions
- `selectedIds`: number[] -- for batch operations
- `expandedId`: number | null -- for detail panel expansion

**Derived state ($derived):**
- `itemCount`: number of items
- `stalledCount`: number of stalled projects (stalledIds.size)

**Methods:**
- `async loadProjects()`: Calls getAllProjects(), stores in items. Then calls getStalledProjects() to get stalled list, builds stalledIds Set from their ids.
- `isStalled(id: number)`: Returns whether id is in stalledIds Set.
- `expandItem(id: number)`: Toggles expandedId (same as ActionState pattern -- if same id, set null, else set id).
- `toggleSelection(id: number)`: Toggle id in selectedIds array (same pattern as ActionState).
- `clearSelection()`: Reset selectedIds to empty.

Export both the class and a singleton instance: `export const projectState = new ProjectState();`

Import types and operations from the appropriate paths:
- GTDItem from '$lib/db/schema' (note: cannot use $lib in .ts files that aren't under routes, use relative path '../db/schema')
- Operations from '../db/operations'

Use relative imports (../db/schema, ../db/operations) NOT $lib aliases, matching the pattern in actions.svelte.ts.
