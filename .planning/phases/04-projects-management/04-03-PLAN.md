   - Heading: "Which project does this belong to?"
   - List existing projects as buttons (same style as context buttons):
     ```
     {#each projects as proj (proj.id)}
       <button onclick={() => { selectedProjectId = proj.id; afterContext = 'project'; step = 'assign-context'; }}>
         {proj.title}
       </button>
     {/each}
     ```
   - A divider, then a "Create new project" section:
     - Input field for new project name, placeholder "New project outcome..."
     - "Create & Link" button that:
       1. Calls `const newId = await addProject(newProjectTitle.trim())`
       2. Sets `selectedProjectId = newId`
       3. Clears newProjectTitle
       4. Sets `afterContext = 'project'` and `step = 'assign-context'`
   - A "Skip (no project)" button that sets selectedProjectId = undefined, afterContext = 'project', step = 'assign-context'
   - Back button calling goBack

6. **Update the project() function:** Change from the current placeholder to actually link to a project:
   ```typescript
   async function project(context?: string) {
     await updateItem(item.id, { type: 'next-action', context, projectId: selectedProjectId });
     storageStatus.recordSave();
     toast.success(selectedProjectId ? 'Added to project as next action.' : 'Added to Next Actions.');
     onProcessed();
   }
   ```

7. **Update goBack():** Add cases for 'select-project' going back to 'delegate-or-defer', and update 'assign-context' to go back to 'select-project' when afterContext is 'project'.

8. **Add GTDItem to imports** from '../db/schema' (needed for projects array typing).

Style all new buttons consistently with the existing button styles in the file.
