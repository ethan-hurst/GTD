## Decisions Made

**Use Plausible's privacy-first visitor hashing approach:**
- Daily-rotating salt (32 random bytes, stored in Netlify Blobs at `salt/YYYY-MM-DD`)
- Hash formula: SHA-256(salt + siteDomain + IP + User-Agent)
- Never persist raw IP or User-Agent - only the hash
- Visitors can't be tracked across days (salt rotation breaks continuity)

**Arrays instead of Sets for visitor tracking:**
- Sets don't serialize to JSON - would lose data on storage/retrieval
- Use array with `.includes()` check for deduplication
- Trade-off: O(n) lookup instead of O(1), acceptable for daily visitor counts

**Pre-aggregate metrics server-side:**
- Store daily totals (pageviews per path, events per type, unique visitors, referrers)
- Avoids querying raw events for dashboard display
- Single Blob per day at `daily/YYYY-MM-DD` key

**sendBeacon with fetch keepalive fallback:**
- sendBeacon preferred (survives page unload, no CORS preflight)
- Fallback to fetch with keepalive: true for older browsers
- Both wrapped in try/catch - analytics must never throw
