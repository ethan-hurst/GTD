In `src/routes/+layout.svelte`:

1. Import the tracker:
   ```
   import { trackPageView } from '$lib/analytics/client';
   ```

2. Add a `$effect` block that tracks page views on route change. Place it AFTER the existing route-based feature tracking `$effect` block (around line 126):
   ```
   // Analytics: track page views on navigation
   $effect(() => {
     const currentPath = $page.url.pathname;
     if (currentPath) {
       trackPageView(currentPath);
     }
   });
   ```

This uses the reactive `$page.url.pathname` store to detect SvelteKit client-side navigation (not just full page loads). The tracker itself handles DNT and SSR safety checks.

Do NOT modify any existing functionality. Just add the import and the $effect block. The trackPageView function is already wrapped in try/catch internally so it cannot break the layout.

<task type="auto">

  <action>
Add analytics events to key user actions. In every case: import `trackEvent` from `'$lib/analytics/client'`, call AFTER the primary action completes, wrap in try/catch if not already wrapped by the tracker.

**ProcessingFlow.svelte** -- Track task creation:
- Find the function/block where items are saved to the database (after `db.items.add()` or similar Dexie call)
- After the successful save, add: `trackEvent('task.created', { type: item.type })` where type is the GTD list type (action, project, waiting, someday, reference, trash)
- Also track task completion: find where items are marked as done/completed and add: `trackEvent('task.completed', { type: item.type })`
- If ProcessingFlow doesn't handle completion, find the component that does (likely in action detail panel or list views) and instrument there instead

**sync.svelte.ts** -- Track sync events:
- In the `sync()` method, after sync starts: `trackEvent('sync.initiated')`
- After sync completes successfully: `trackEvent('sync.completed', { itemCount: items.length, duration: Math.round(elapsed) })` where elapsed is ms from start to finish
- Add `const syncStart = Date.now()` before the sync operation and calculate duration after

**review.svelte.ts** -- Track weekly review:
- When review starts (user begins wizard): `trackEvent('review.started')`
- When review completes (all steps done): `trackEvent('review.completed')`
- Find the start/complete methods or state transitions and add the calls there

**onboarding.svelte.ts** -- Track onboarding funnel:
- When onboarding starts: `trackEvent('onboarding.started')`
- When onboarding completes: `trackEvent('onboarding.completed')`
- When onboarding is skipped: `trackEvent('onboarding.skipped')`
- Find the `startOnboarding()`, `completeOnboarding()`, and skip methods

IMPORTANT patterns (from reverted Phase 08.7 decisions):
- Analytics calls placed AFTER primary action completes (non-blocking, fail-safe)
- Sync metrics include itemCount and duration for operational insight
- Single taskCompleted event for all completion paths (simpler than per-category events)
- All trackEvent calls are inherently non-blocking (sendBeacon) and fail-safe (try/catch inside client.ts)
  </action>

  </verify>
  <done>
Custom events fire for: task creation (with type), task completion (with type), sync initiated/completed (with itemCount + duration), review started/completed, onboarding started/completed/skipped. All non-blocking, all fail-safe.
  </done>
