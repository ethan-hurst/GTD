Create `src/routes/analytics/+page.svelte`:

**Authentication flow:**
- On mount, prompt user for password using `window.prompt('Enter analytics password:')` (simple, no need for a full auth form for an admin tool)
- Store password in a local variable (not persisted -- re-prompt on each visit for security)
- If user cancels prompt, show "Access denied" message
- Build Basic auth header: `'Basic ' + btoa('admin:' + password)`

**Data fetching:**
- Fetch from `/.netlify/functions/analytics-query?days=30` with the auth header
- Handle 401 (wrong password) by showing error and re-prompting
- Handle 503 (not configured) by showing setup instructions
- Store response data in reactive state

**Dashboard UI (using existing Tailwind patterns from the app):**

Layout: full-width, padded with p-6 like other pages.

1. **Header section:**
   - Title: "Analytics Dashboard"
   - Subtitle showing date range: "Last 30 days"
   - Period selector buttons: 7d, 30d, 90d (re-fetch on click)

2. **Summary cards row** (grid grid-cols-2 tablet:grid-cols-4 gap-4):
   - Total Pageviews (with formatted number)
   - Unique Visitors (with formatted number)
   - Top Event (name + count)
   - Period (start - end dates)

3. **Visitors Over Time chart** (line chart):
   - Lazy-load Chart.js: `const { Chart } = await import('chart.js/auto')` inside `onMount`
   - Line chart with dates on x-axis, unique visitors on y-axis
   - Second dataset: daily pageviews (different color)
   - Use blue (#3b82f6) for visitors, green (#10b981) for pageviews
   - Responsive, maintain aspect ratio on mobile
   - Canvas element with bind:this

4. **Top Pages table** (styled like existing list views):
   - Columns: Path, Views
   - Show top 10 pages from API response
   - Sorted by views descending

5. **Event Breakdown table:**
   - Columns: Event, Count
   - Show all tracked events from API response
   - Sorted by count descending

6. **Referrers table** (if any data):
   - Columns: Source, Count
   - Show top referrers

**Dark mode support:** Use existing Tailwind dark: classes matching the app's style (bg-white dark:bg-gray-900 for cards, etc.)

**Loading state:** Show "Loading analytics..." with a simple spinner while fetching.

**Error state:** Show error message with retry button.

Do NOT use `+page.server.ts` -- this app has `ssr = false` globally (see +layout.ts). All data fetching happens client-side.

IMPORTANT: Chart.js must be dynamically imported (`await import('chart.js/auto')`) inside `onMount` to code-split it from the main bundle. Users who never visit /analytics should not download Chart.js.
