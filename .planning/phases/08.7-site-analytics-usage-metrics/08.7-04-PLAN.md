3. Create `src/routes/analytics/+page.svelte`:

   This is a standalone dashboard page -- it does NOT use the app's sidebar layout (it's for team/admin use, not GTD workflow). Style it independently with Tailwind.

   **State management (Svelte 5 $state runes):**
   - `let password = $state('')` -- user-entered password
   - `let authenticated = $state(false)` -- whether auth succeeded
   - `let loading = $state(false)` -- loading indicator
   - `let error = $state('')` -- error message
   - `let data = $state(null)` -- analytics response data
   - `let days = $state(30)` -- selected date range (7, 30, 90)
   - `let chartCanvas: HTMLCanvasElement` -- bound canvas element
   - `let chartInstance: any = null` -- Chart.js instance ref for cleanup

   **Password prompt UI:**
   - Show a centered card with password input and "View Analytics" button when `!authenticated`
   - On submit, call `fetchAnalytics()` which sets `loading=true`, fetches from API, sets `authenticated=true` on success
   - Store password in memory for subsequent fetches (date range changes)
   - On 401 response, show "Invalid password" error, keep prompt visible

   **Data fetching function `fetchAnalytics()`:**
   ```
   const response = await fetch(`/.netlify/functions/analytics-query?days=${days}`, {
     headers: {
       'Authorization': `Basic ${btoa(`admin:${password}`)}`
     }
   });
   ```
   - On 401: set error = 'Invalid password'
   - On 503: set error = 'Analytics not configured. Set ANALYTICS_PASSWORD in Netlify.'
   - On success: parse JSON, set data, set authenticated = true

   **Dashboard layout (when authenticated):**

   A. **Header bar:**
   - Title "Analytics Dashboard" left-aligned
   - Date range selector (7d / 30d / 90d) as segmented buttons, right-aligned
   - Changing date range calls `fetchAnalytics()` again with new days value
   - Show `generatedAt` timestamp below title in gray text

   B. **Summary cards row (grid, 3 columns on desktop, 1 on mobile):**
   - Card 1: "Total Pageviews" -- `data.summary.totalPageviews.toLocaleString()`
   - Card 2: "Unique Visitors" -- `data.summary.totalUniqueVisitors.toLocaleString()`
   - Card 3: "Events Tracked" -- sum of all `data.summary.topEvents[].count` formatted

   C. **Line chart (visitors + pageviews over time):**
   - Canvas element bound to `chartCanvas`
   - Dynamically import Chart.js in `$effect` when data changes:
     ```
     const ChartJS = (await import('chart.js/auto')).default;
     ```
   - If `chartInstance` exists, call `.destroy()` before creating new chart (prevents memory leak)
   - Two datasets:
     - "Unique Visitors" -- blue line (rgb(59, 130, 246)), data from `data.summary.dailyMetrics.map(d => d.uniqueVisitors)`
     - "Pageviews" -- green line (rgb(34, 197, 94)), data from `data.summary.dailyMetrics.map(d => d.pageviews)`
   - Labels: `data.summary.dailyMetrics.map(d => d.date)`
   - Options: `responsive: true, maintainAspectRatio: false`, tension: 0.3 for smooth curves
   - Chart container: `height: 300px` wrapper div

   D. **Ranked lists (grid, 3 columns on desktop, 1 on mobile):**
   - "Top Pages" -- ordered list from `data.summary.topPages`, showing path and view count
   - "Top Events" -- ordered list from `data.summary.topEvents`, showing event name and count
   - "Referrers" -- ordered list from `data.summary.referrers`, showing source and count
   - Each list: max 10 items, styled as table rows with alternating backgrounds
   - Show "No data yet" if array is empty

   E. **Footer:**
   - "Data retention: 13 months" note
   - "Privacy: No cookies, no PII, daily-rotating visitor hashes"

   **Styling:**
   - Use Tailwind utility classes throughout
   - Background: `bg-gray-50 dark:bg-gray-900 min-h-screen`
   - Cards: `bg-white dark:bg-gray-800 rounded-xl shadow-sm p-6`
   - Text: `text-gray-900 dark:text-gray-100` for headings, `text-gray-500 dark:text-gray-400` for secondary
   - The page should look clean and professional as a standalone admin tool
   - Support dark mode (this project uses class-based dark mode)

   **Chart.js cleanup:**
   - Use Svelte 5's `$effect` with return cleanup function to destroy chart on unmount:
     ```
     $effect(() => {
       // ... chart creation code ...
       return () => { if (chartInstance) chartInstance.destroy(); };
     });
     ```

   **Important implementation notes:**
   - Dynamic import `chart.js/auto` inside the effect -- do NOT put in top-level imports (code splitting)
   - The page is self-contained -- it imports nothing from the GTD app's stores or components
   - Password is held in memory only (not localStorage, not cookies) for security
   - All number displays use `.toLocaleString()` for formatting
