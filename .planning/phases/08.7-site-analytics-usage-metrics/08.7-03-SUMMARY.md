## Decisions Made

**1. Basic auth with ANALYTICS_PASSWORD env var**
- Rationale: Simple, secure protection for query endpoint without requiring user management
- Alternative considered: API keys - rejected for added complexity

**2. Opportunistic cleanup on each query**
- Rationale: Ensures retention policy enforced without separate cron job
- Fire-and-forget pattern prevents cleanup failures from breaking queries

**3. Default 30-day range, max 395 days**
- Rationale: 30 days covers most dashboard use cases, 395 days (~13 months) matches retention policy
- Prevents accidental over-fetching while allowing full historical access

**4. Unique visitors not deduplicated across days**
- Rationale: Daily hash rotation means same visitor gets different IDs each day (privacy feature from Plan 01)
- totalUniqueVisitors = sum of daily unique counts, not global unique count
- Documented in code comments for future maintainers

**Environment variable required for analytics query API:**

Add to `.env` or Netlify environment variables:


**Verification:**
```bash

curl https://your-site.netlify.app/.netlify/functions/analytics-query

# With auth (should return JSON)
curl -u admin:your_password https://your-site.netlify.app/.netlify/functions/analytics-query?days=7
```
