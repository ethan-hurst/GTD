  <files>
    netlify/functions/analytics-utils.ts

  <action>
Add the following functions to the existing `netlify/functions/analytics-utils.ts` (created in Plan 01). Do NOT modify existing functions -- only append new exports.

`cleanupOldData(store): Promise<number>`:
- Calculate cutoff date: 13 months ago from today (YYYY-MM-DD format)
- List all blobs with prefix `daily/` using `store.list({ prefix: 'daily/' })`
- For each blob key, compare the date portion (after `daily/`) with cutoff
- Delete blobs older than cutoff: `await store.delete(blob.key)`
- Also clean up old salts: list prefix `salt/`, delete any with date < cutoff
- Return count of deleted entries
- Log cleanup activity: `console.log('Analytics cleanup: deleted ${count} old entries')`
- This is idempotent and safe to run multiple times (research recommendation)

`aggregateMetrics(dailyData: DailyAggregate[]): AggregatedMetrics`:
- Input: array of daily aggregates
- Output: `AggregatedMetrics` type with:
  - `totalPageviews: number` (sum of all pageview counts across all days)
  - `totalUniqueVisitors: number` (sum of daily unique visitors -- note: not deduplicated across days, that's by design with daily hash rotation)
  - `dailyMetrics: { date: string; pageviews: number; uniqueVisitors: number; events: number }[]` (one entry per day, sorted by date ascending)
  - `topPages: { path: string; views: number }[]` (top 10 pages by total views, sorted descending)
  - `topEvents: { event: string; count: number }[]` (top 10 events by count, sorted descending)
  - `referrers: { source: string; count: number }[]` (top 10 referrers, sorted descending)

Export the `AggregatedMetrics` type for use by the query function.

Pageviews per day = sum of all values in `dailyAggregate.pageviews` object.
Events per day = sum of all values in `dailyAggregate.events` object.

For topPages: merge all dailyAggregate.pageviews into a single Record<string, number>, sort by value descending, take top 10.
For topEvents: same pattern with dailyAggregate.events.
For referrers: same pattern with dailyAggregate.referrers.

<task type="auto">
  <name>Task 2: Create analytics query Netlify Function with auth and date range</name>
  <files>
    netlify/functions/analytics-query.mts

  <action>
Create `netlify/functions/analytics-query.mts`:

1. Accept GET only (return 405 for other methods)

2. Authentication via `ANALYTICS_PASSWORD` environment variable:
   - Check `Authorization` header for `Basic` scheme
   - Decode base64 credentials, extract password (username can be anything, e.g., 'admin')
   - Compare password against `process.env.ANALYTICS_PASSWORD` (or Netlify.env.get if available)
   - If no ANALYTICS_PASSWORD is set, return 503 with `{ error: 'Analytics not configured' }`
   - If auth fails, return 401 with `WWW-Authenticate: Basic realm="Analytics"` header

3. Parse query params for date range:
   - `days` param (default: 30, max: 395 which is ~13 months)
   - Calculate date array: last N days as YYYY-MM-DD strings

4. Fetch daily aggregates:
   - `const store = getStore('analytics')`
   - For each date in range, fetch `daily/${date}` using `getDailyAggregate(store, date)` from analytics-utils
   - Use `Promise.all()` for parallel fetching (performance)

5. Run cleanup opportunistically:
   - Call `cleanupOldData(store)` after fetching data (non-blocking, don't await -- fire and forget)
   - This ensures cleanup runs at least whenever someone views the dashboard
   - Wrap in try/catch so cleanup failure doesn't break query

6. Aggregate via `aggregateMetrics(dailyData)` from analytics-utils

7. Return 200 with JSON:
   ```
   {
     period: { start: firstDate, end: lastDate, days: N },
     summary: aggregatedMetrics,
     generatedAt: new Date().toISOString()
   }
   ```

8. Set CORS headers: `Access-Control-Allow-Origin: *` (the dashboard may be on the same domain, but this keeps it flexible)

9. Error handling: try/catch around everything, return 500 on unexpected errors

Follow sync-push.mts patterns for Response construction. Import from '@netlify/blobs' and '@netlify/functions'.
