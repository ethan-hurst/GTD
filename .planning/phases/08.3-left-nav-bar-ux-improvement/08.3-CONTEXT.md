# Phase 08.3: Left Nav Bar UX Improvement - Context

**Gathered:** 2026-01-31
**Status:** Ready for planning

<domain>
## Phase Boundary

Redesign the left sidebar navigation to eliminate the scrollbar on a 14" MacBook Pro at full screen, while preserving all existing functionality: nav links (Inbox, Next Actions, Projects, Waiting For, Someday/Maybe, Weekly Review, Calendar, Settings), context list with filtering, badges/counts, theme toggle, and active state indicators.

</domain>

<decisions>
## Implementation Decisions

### Scroll elimination strategy
- Claude's discretion on approach — collapsible sections, tighter spacing, or a combination
- Must work without scrollbar on 14" MBP at full screen (browser full-height, ~900px viewport)
- If overflow still occurs on very short viewports or many contexts added, Claude decides fallback (hidden scrollbar or auto-collapse)
- Sidebar width is flexible — Claude can adjust from current 240px if it helps
- Nav item priority: Claude decides based on GTD methodology what can be de-emphasized

### Context list treatment
- User has 3-5 contexts typically — not a heavy list
- Clicking a context filters Next Actions inline — keep this behavior
- Inline "add context" form is rarely used — can be de-emphasized or moved
- Claude decides presentation: collapsed by default, always visible, or popover — whatever eliminates the scroll issue

### Visual hierarchy & density
- Claude decides density — balance between eliminating scroll and looking good
- Current horizontal line separators between groups — Claude decides whether to keep, replace with section headers, or use spacing
- Icons: no strong opinions — keep or adjust as needed
- No specific app reference — just make it work well and look professional

### Responsive behavior
- Primary target: 14" MacBook Pro, full-screen browser
- One consistent layout that works on 14" and scales up — no need for adaptive viewport breakpoints
- Sidebar collapse toggle: nice to have — a way to hide the sidebar entirely for more content space

### Claude's Discretion
- Overall approach to scroll elimination (spacing, grouping, collapsing, or combination)
- Sidebar width adjustment
- Nav item prioritization and grouping strategy
- Context list presentation pattern
- Density and spacing decisions
- Separator/grouping visual treatment
- Icon choices
- Whether responsive adaptations are worthwhile
- Fallback behavior for edge-case overflow
- Sidebar collapse toggle implementation (if included)

</decisions>

<specifics>
## Specific Ideas

- Scrollbar appears on 14" MBP even at full screen — current layout is too tall
- User wants to deploy and use this as a hosted site — needs to look polished
- Inline "add context" form is rarely used and could be hidden behind an icon or moved to Settings
- Sidebar collapse toggle would be a nice addition for extra content space

</specifics>
