<objective>
Eliminate the sidebar scrollbar on 14-inch MacBook Pro by reducing nav item spacing, replacing separators with visual grouping, and making the context list collapsible via native HTML `<details>`/`<summary>`.

Purpose: The sidebar currently overflows at ~900px viewport height because the context list takes ~200px when expanded. By reducing padding (py-2 to py-1.5, saving ~64px), removing separators in favor of spacing (saving ~20px), and making the context list collapsible by default (saving ~150px), the total height drops well within the viewport budget.

Output: Redesigned Sidebar.svelte with tighter spacing, ContextList.svelte wrapped in native collapsible `<details>` element, and supporting CSS in app.css.
</objective>

Redesign the sidebar navigation in Sidebar.svelte with the following changes:

**Spacing reduction (all nav links):**
- Change all nav link padding from `py-2` to `py-1.5` (saves ~4px per item, ~32px total across 8 items)
- Change nav container padding from `p-4` to `px-3 py-3` for slightly tighter horizontal and vertical padding

**Replace separator lines with spacing:**
- Remove all 5 `<div class="border-t ...">` separator elements
- Replace with semantic grouping using `<div class="mt-3">` before each group (saves ~20px total since separators had `my-2` = 16px margin plus the border itself)
- Group the nav items logically:
  - **Capture group:** Inbox (stands alone at top, primary GTD entry point)
  - **Action group (mt-3):** Next Actions + ContextList (indented under Next Actions)
  - **Organize group (mt-3):** Projects, Waiting For, Someday/Maybe
  - **Review group (mt-3):** Weekly Review, Calendar
  - **System group (mt-3):** Settings

**Header optimization:**
- Reduce header padding from `py-4` to `py-3` (saves ~8px)
- Keep the GTD logo animation and styling intact

**Footer optimization:**
- Reduce footer padding from `p-4` to `px-3 py-2` (saves ~8px)

**Nav overflow strategy:**
- Keep `overflow-y-auto` on the nav element as a safety net for very short viewports or many contexts
- The goal is that it never activates on 14" MBP, but it's there if needed

**FeatureHint wrapper note:**
- The FeatureHint wrapper uses `class="relative inline-block"` which should not affect vertical spacing. Keep all FeatureHint wrappers in place.

```svelte
            ({actionState.contexts.length})
        </span>
    </span>
    <span class="flex items-center gap-1">
        <!-- Add context button (keep existing) -->
        <button onclick|stopPropagation={handleAddClick} ...existing classes... title="Add context">
            <!-- existing + icon SVG -->
        </button>
        <!-- Chevron indicator -->
        <svg class="w-4 h-4 text-gray-400 transition-transform duration-150 context-chevron" fill="none" stroke="currentColor" viewBox="0 0 24 24">
            <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7" />
        </svg>
    </span>
</summary>
```

3. Wrap the remaining content (All button, context items loop, add form) in a `<div>` inside `<details>`:
```svelte
<div class="space-y-1">
    <!-- All view button -->
    <!-- Context items each loop -->
    <!-- Add context form -->
</div>
```

4. Important: The `<details>` element does NOT have the `open` attribute, so it starts collapsed by default.

5. The "Add context" button on the summary must use event.stopPropagation() on click to prevent toggling the details open/closed when clicking the + button. Use Svelte event modifier: `onclick={(e) => { e.stopPropagation(); handleAddClick(); }}`

6. When the + button is clicked and isAddingContext becomes true, programmatically open the details element so the form is visible. Add this logic:
```svelte
function handleAddClick() {
    isAddingContext = true;
    // Ensure details is open when adding
    const detailsEl = document.querySelector('.context-details');
    if (detailsEl && !detailsEl.hasAttribute('open')) {
        detailsEl.setAttribute('open', '');
    }
    setTimeout(() => inputElement?.focus(), 50);
}
```
