<task type="auto">
  <name>Task 1: Test sync-push validation and response contracts</name>
  <files>netlify/functions/sync-push.test.ts</files>
  <action>
Create `netlify/functions/sync-push.test.ts` that tests the sync-push handler by mocking `@netlify/blobs`.

1. Mock `@netlify/blobs` at the module level using `vi.mock('@netlify/blobs', ...)`. The mock should return a `getStore` function that returns a fake store object with a `set` method (use `vi.fn()` that resolves to undefined). Keep a reference to the mock `set` fn so tests can assert on calls.

2. Import the default export from `./sync-push.mts` (the handler function).

3. Create a helper `mockContext()` that returns a minimal object satisfying the Netlify `Context` type (can be an empty object cast as `any` — the functions don't use context fields).

4. Create a helper `makeRequest(method, body?)` that constructs a `new Request('http://localhost/.netlify/functions/sync-push', { method, body: JSON.stringify(body), headers: { 'Content-Type': 'application/json' } })`.

5. Write these test cases in a `describe('sync-push')` block:

   **Method validation:**
   - GET request returns 405 with `{ error: 'Method not allowed' }`
   - PUT request returns 405

   **Field validation:**
   - Missing deviceId returns 400 with error message containing 'deviceId'
   - Missing encryptedBlob returns 400 with error message containing 'encryptedBlob'
   - deviceId as number (non-string) returns 400
   - encryptedBlob as number (non-string) returns 400

   **Payload size:**
   - encryptedBlob over 10MB returns 413 with error containing 'too large'

   **Success path:**
   - Valid POST with deviceId + encryptedBlob returns 200 with `{ success: true, timestamp: ... }`
   - Verify `store.set` was called with (deviceId, encryptedBlob)
   - Valid POST with contentHash also stores hash: verify `store.set` called twice (once for blob, once for deviceId + '-hash')
   - Valid POST without contentHash: verify `store.set` called once (only blob, no hash)

   **Error handling:**
   - If `store.set` throws, returns 500 with `{ error: 'Internal server error' }`

6. Each test should call `handler(request, mockContext())` and assert on `response.status` and `await response.json()`.

IMPORTANT: The Vitest config uses browser mode. If the `netlify/functions/` directory is outside the Vitest include pattern, you may need to check the vitest.config.ts include glob. If it only covers `src/**/*.test.ts`, either:
- Add `'netlify/functions/**/*.test.ts'` to the include array in vitest.config.ts, OR
- Update the existing glob to also match netlify/functions paths.
Check the actual vitest.config.ts include pattern and update if needed.

Create test files for both sync-pull and sync-check. These two functions share the same validation pattern (GET-only, deviceId query param, 64-char hex format check).

**netlify/functions/sync-pull.test.ts:**

1. Mock `@netlify/blobs` with `getStore` returning a fake store with a `get` method (vi.fn()). Keep a reference to configure return values per test.

2. Import the default export from `./sync-pull.mts`.

3. Create helpers: `mockContext()` (same as Task 1), `makeRequest(method, params?)` that builds `new Request('http://localhost/.netlify/functions/sync-pull' + (params ? '?' + new URLSearchParams(params).toString() : ''))` with the given method.

4. Write test cases in `describe('sync-pull')`:

   **Method validation:**
   - POST request returns 405
   - PUT request returns 405

   **deviceId validation:**
   - Missing deviceId returns 400 with error containing 'deviceId'
   - deviceId with invalid format (not 64-char hex, e.g., 'abc123') returns 400 with error containing 'format'

   **Not found:**
   - Valid deviceId but store.get returns null: returns 200 with `{ found: false, message: ... }`

   **Found:**
   - Valid deviceId and store.get returns 'encrypted-data-here': returns 200 with `{ found: true, encryptedBlob: 'encrypted-data-here', timestamp: ... }`
   - Verify `store.get` was called with the deviceId and `{ type: 'text' }`

   **Error handling:**
   - If `store.get` throws, returns 500

**netlify/functions/sync-check.test.ts:**

1. Same mock pattern as sync-pull.

2. Import the default export from `./sync-check.mts`.

3. Same helpers pattern.

4. Write test cases in `describe('sync-check')`:

   **Method validation:**
   - POST request returns 405

   **deviceId validation:**
   - Missing deviceId returns 400
   - Invalid format returns 400

   **Not found:**
   - Valid deviceId but store.get(deviceId + '-hash') returns null: returns 200 with `{ found: false }`

   **Found:**
   - Valid deviceId and store.get returns 'abc123hash': returns 200 with `{ found: true, hash: 'abc123hash' }`
   - Verify `store.get` was called with `deviceId + '-hash'` and `{ type: 'text' }`

   **Error handling:**
   - If `store.get` throws, returns 500

Use a valid 64-char hex deviceId in tests: `'a'.repeat(64)` (a string of 64 'a' characters is valid hex).
