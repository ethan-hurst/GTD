<task type="auto">
  <name>Task 1: Install test dependencies and configure Vitest with Browser Mode</name>
  <files>package.json, vitest.config.ts, src/tests/setup.ts</files>
  <action>
1. Install all test dependencies:
   ```
   npm install -D vitest @vitest/browser @vitest/browser-playwright vitest-browser-svelte playwright fake-indexeddb @vitest/coverage-v8
   ```
   Then install Playwright chromium browser:
   ```
   npx playwright install chromium
   ```

2. Create `vitest.config.ts` as a SEPARATE file from vite.config.ts (do NOT modify vite.config.ts). Vitest will automatically merge with vite.config.ts. Configure:
   - Import `defineConfig` from 'vitest/config'
   - Import `svelte` from '@sveltejs/vite-plugin-svelte'
   - Import `playwright` from '@vitest/browser-playwright'
   - Set `resolve.conditions: ['browser']` when `process.env.VITEST` is set (needed for Svelte browser imports)
   - Enable browser mode with `browser.enabled: true`
   - Set provider to `playwright()` function call
   - Set instances to `[{ browser: 'chromium' }]`
   - Set `headless: true` for CI compatibility
   - Set setupFiles to `['./src/tests/setup.ts']`
   - Set `globals: true` for describe/it/expect
   - Configure coverage with provider 'v8', reporters ['text', 'html'], and exclude node_modules, tests, configs
   - Set `include` pattern to match `**/*.test.ts` files
   - IMPORTANT: Do NOT include the sveltekit() plugin in vitest.config.ts - use the raw svelte() plugin instead. SvelteKit plugin interferes with Vitest browser mode.
   - IMPORTANT: Do NOT add `@tailwindcss/vite` plugin - it's not needed for tests and may cause issues.

3. Create `src/tests/setup.ts`:
   - Import 'fake-indexeddb/auto' (polyfills global indexedDB)
   - Import `beforeEach` from 'vitest'
   - Import `IDBFactory` from 'fake-indexeddb'
   - In beforeEach, reset indexedDB: `indexedDB = new IDBFactory()`
   - Add a TypeScript comment explaining why reset is needed (test isolation)

4. Update `package.json` scripts - ADD these scripts (keep existing ones):
   - `"test": "vitest"` (watch mode for development)
   - `"test:run": "vitest run"` (single run for CI)
   - `"test:coverage": "vitest run --coverage"` (with coverage report)
   - `"test:e2e": "playwright test"` (E2E tests)
   - `"test:e2e:ui": "playwright test --ui"` (E2E with UI mode)
  </action>
