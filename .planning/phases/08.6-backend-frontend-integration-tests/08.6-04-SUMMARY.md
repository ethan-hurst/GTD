## Decisions Made

1. **Use path aliases instead of relative imports in tests**
   - Added `$lib` → `./src/lib` and `$app` → `./src/tests/mocks/$app` aliases to vitest.config.ts
   - Matches SvelteKit convention, enables component imports to resolve correctly

2. **Create custom $app mocks instead of using @sveltejs/kit mocks**
   - Browser mode doesn't play well with SvelteKit's built-in test mocks
   - Custom mocks provide minimal API surface needed for component rendering

3. **Exclude playwright from optimizeDeps**
   - Vite tried to bundle playwright-core dependencies (chromium-bidi, fsevents.node)
   - Exclusion prevents bundling errors while keeping browser mode functional

4. **Use Date objects (not timestamps) in mock data**
   - GTDItem schema uses `created: Date` not `created: number`
   - formatRelativeTime() expects Date object with .getTime() method
   - Tests initially failed with "date.getTime is not a function" until fixed
