**1. Deferred CRUD form submission tests**

- **Decision:** Defer full CRUD flow tests to avoid blocking E2E infrastructure completion

- **Decision:** Test uses conditional logic to verify either empty state OR item count display
- **Rationale:** Test reliability across different data scenarios (fresh DB vs seeded DB)

**3. Viewport-conditional navigation assertions**
- **Context:** Mobile uses drawer navigation, desktop uses sidebar
- **Decision:** Smoke tests check viewport size and adapt assertions accordingly
- **Rationale:** Single test suite validates both responsive layouts without duplication
