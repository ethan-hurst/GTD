## Decisions Made

**Separate vitest configs instead of workspace:**
- Considered vitest workspace feature for browser + node environments
- Chose separate configs for simpler mental model and clearer separation
- Browser tests use `vitest.config.ts`, Node.js tests use `vitest.node.config.ts`
- Trade-off: Two configs to maintain vs. single workspace config
- Rationale: Simpler for developers to understand and easier to run independently

**Mock strategy:**
- Mock @netlify/blobs at module level with vi.mock
- Return factory function for getStore to allow per-test customization
- Reset mocks in beforeEach for test isolation
- Pattern works well for serverless function testing

**Test organization:**
- Group tests by HTTP status code path (405 → 400 → 413 → 200 → 500)
- Verify both status code AND JSON body shape in each test
- Helper functions (mockContext, makeRequest) reduce boilerplate
- Pattern is clear and maintainable

## Issues Encountered

**Initial test failure with browser mode:**
- Problem: Netlify Functions tests tried to run in browser mode (Playwright)
- Browser mode bundler failed on Node.js-specific modules (chromium-bidi, fsevents.node)
- Root cause: vitest.config.ts forces browser mode globally
- Solution: Created vitest.node.config.ts with `environment: 'node'`
- Modified vitest.config.ts include pattern to exclude netlify/functions
- Tests now run cleanly in Node.js environment
- Pattern established for future backend testing

**No other issues encountered** - mocking strategy worked on first attempt, all tests passed immediately.
