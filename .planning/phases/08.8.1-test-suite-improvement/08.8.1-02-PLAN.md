import { describe, it, expect } from 'vitest';

**Test cases for parseICS:**

describe('parseICS - basic events'):
- Parses single VEVENT with SUMMARY, DTSTART, DTEND
  ```
  const ics = `BEGIN:VCALENDAR
  VERSION:2.0
  PRODID:-//Test//Test//EN
  BEGIN:VEVENT
  DTSTART:20260201T100000Z
  DTEND:20260201T110000Z
  SUMMARY:Team Meeting
  END:VEVENT
  END:VCALENDAR`;
  ```
  Verify: result[0].title === 'Team Meeting', startTime matches, endTime matches
- Parses multiple VEVENTs from single calendar
- Sets default source to 'ics-import'
- Accepts custom source parameter
- Event without SUMMARY gets title '(No title)'
- Skips events without DTSTART (logs warning)

describe('parseICS - all-day events'):
- Detects all-day event (DATE type DTSTART, no time component)
  Use DTSTART;VALUE=DATE:20260201 format
- All-day event sets allDay to true
- Missing DTEND on all-day event defaults to +1 day

describe('parseICS - missing DTEND fallback'):
- Timed event without DTEND gets +1 hour duration
- All-day event without DTEND gets +1 day duration

describe('parseICS - optional properties'):
- Extracts LOCATION when present
- Extracts DESCRIPTION as notes when present
- Sets location to undefined when not present
- Sets notes to undefined when not present

describe('parseICS - recurrence'):
- Extracts RRULE string from recurring event
  Use: `RRULE:FREQ=DAILY;COUNT=5`
  Verify: result[0].rrule === 'FREQ=DAILY;COUNT=5'
- Extracts EXDATE (exception dates) as ISO strings
- rrule is undefined for non-recurring events
- exceptionDates is undefined when no EXDATE present

describe('parseICS - error handling'):
- Throws descriptive error for completely invalid input ('not ics data')
- Throws for malformed VCALENDAR (missing END tag)

**Test cases for validateICS:**

describe('validateICS'):
- Returns true for valid ICS starting with BEGIN:VCALENDAR
- Returns true with leading whitespace (trimmed)
- Returns false for empty string
- Returns false for random text
- Returns false for JSON content
- Returns false for HTML content

**IMPORTANT about ICS format in test strings:** Use template literals. Each property must be on its own line. Lines must NOT have leading whitespace (ICS format is strict about line starts). Use this pattern:
```typescript
const ics = [
  'BEGIN:VCALENDAR',
  'VERSION:2.0',
  'PRODID:-//Test//EN',
  'BEGIN:VEVENT',
  'DTSTART:20260201T100000Z',
  'DTEND:20260201T110000Z',
  'SUMMARY:Test Meeting',
  'END:VEVENT',
  'END:VCALENDAR'
].join('\r\n');
```
Using `\r\n` line endings and array join is safest to avoid whitespace issues with template literals and editor auto-indentation.
