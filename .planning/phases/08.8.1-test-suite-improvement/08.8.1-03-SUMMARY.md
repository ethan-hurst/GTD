# Phase 08.8.1 Plan 03: Analytics Netlify Function Tests Summary

Fixed broken vitest.node.config.ts include pattern and added 45 tests across analytics-track, analytics-query, and analytics-utils -- bringing Node.js test suite from 29 to 74 tests with zero failures.

## What Was Done

### Task 1: Fix vitest.node.config.ts include pattern

Fixed the include pattern from `netlify/functions/**/*.test.ts` (matched zero files) to `tests/netlify-functions/**/*.test.ts` (matches all test files). The config now correctly discovers all Node.js function tests. Verified with `npx vitest run --config vitest.node.config.ts` finding 3 existing sync test files with 29 passing tests.

**Commit:** `350e35d` -- fix(08.8.1-03): fix vitest.node.config.ts include pattern

### Task 2: Create analytics function tests (track, query, utils)

Created three test files following the established sync-push.test.ts pattern:

**analytics-track.test.ts (17 tests):**
- Method validation (405 for GET, PUT)
- DNT header respect (tracked:false, no storage calls)
- Field validation (missing event, non-string event, missing url, non-string url)
- Happy path (pageview returns tracked:true)
- Visitor ID generation (calls generateVisitorId with request and 'gtd-app')
- Pageview counting (new and incremented)
- Event tracking for all event types
- Visitor deduplication (no duplicate in array, new visitors added)
- Referrer tracking (from properties, empty string ignored)
- Error handling (500 on internal error)

**analytics-query.test.ts (13 tests):**
- Method validation (405 for POST)
- Environment check (503 when ANALYTICS_PASSWORD not set)
- Authentication (401 without auth header, 401 with non-Basic auth, 401 with wrong password)
- Happy path (200 with aggregated metrics)
- Date range defaults (30 days, custom days param, 395 cap)
- Data flow (aggregateMetrics called with fetched data)
- Opportunistic cleanup (cleanupOldData called)
- Response format (period start/end dates, CORS header)

**analytics-utils.test.ts (15 tests):**
- aggregateMetrics: zero totals for empty input, pageview summation, visitor summation (not deduplicated), date sorting, top-10 pages/events/referrers with sorting, cross-day page merging
- getDailyAggregate: returns existing data from store, returns default empty aggregate
- updateDailyAggregate: stores JSON with metadata
- cleanupOldData: deletes old daily entries, deletes old salt entries, keeps recent entries, returns total count

## Decisions Made

1. **Mock analytics-utils at module level for handler tests** -- Rather than letting analytics-utils run against a mocked @netlify/blobs store, the handler tests mock the utility functions directly. This isolates handler logic and avoids complex mock store setup for functions that internally call getStore.

2. **Top-level mock function references for utils tests** -- The analytics-utils.test.ts uses top-level `mockGet`, `mockSet`, `mockList`, `mockDelete` references to allow direct assertion on store method calls, following a pattern that avoids re-extracting mocks from the getStore return value.
