Create the directory `src/lib/data/` and the file `src/lib/data/changelog.ts` with:

1. **ChangelogEntry interface** with fields:
   - `id: string` (unique ID for last-seen tracking, use date-based format like "2026-02-03")
   - `date: string` (ISO date: "2026-02-03")
   - `version?: string` (optional semver)
   - `categories` object with optional arrays for: `added`, `improved`, `fixed`, `changed`, `deprecated`, `removed`, `security`

   - removed: red (bg-red-500, text-red-700 dark:text-red-400)
   - security: dark red (bg-red-600, text-red-800 dark:text-red-300)

- Added: Wholly new feature

- Fixed: Bug correction
- Changed: Modification to existing behavior

<task type="auto">
  <name>Task 2: Create changelog page at /changelog</name>
  <files>src/routes/changelog/+page.svelte</files>
  <action>
Create `src/routes/changelog/+page.svelte` following existing app page patterns (reference Settings page for layout conventions).

**Page structure:**

1. **Script section** (use `lang="ts"` and Svelte 5 patterns):
   - Import `changelog`, `getCategoryStyle`, `STORAGE_KEY` from `$lib/data/changelog`
   - Import `onMount` from `svelte`
   - Create `lastSeenId` state with `$state('')`
   - On mount: read `localStorage.getItem(STORAGE_KEY)` into lastSeenId
   - On mount: set a 2-second timeout to mark the latest entry as seen via `localStorage.setItem(STORAGE_KEY, changelog[0].id)` and update lastSeenId. Clean up timeout on unmount.
   - Create `formatDate(dateStr: string)` function using `new Date(dateStr + 'T00:00:00').toLocaleDateString('en-US', { year: 'numeric', month: 'long', day: 'numeric' })` -- add T00:00:00 to avoid timezone offset issues with date-only strings.
   - Create `isNewEntry(entryId: string)` function that returns true if the entry is newer than lastSeenId (entry's index is less than lastSeenId's index in the array, or lastSeenId not found).
   - Define `categoryOrder` array: `['added', 'improved', 'fixed', 'changed', 'deprecated', 'removed', 'security']`
   - Define `categoryLabels` record mapping each key to its display label (Added, Improved, Fixed, Changed, Deprecated, Removed, Security)

2. **Page layout** (match existing app patterns):
   - Outer container: `max-w-4xl mx-auto p-4 tablet:p-6` (matches app page pattern from research context)
   - Header: h1 "What's New" with `text-2xl tablet:text-3xl font-bold text-gray-900 dark:text-gray-100 mb-2`
   - Subtitle: `text-sm tablet:text-base text-gray-600 dark:text-gray-400 mb-8` with text "Recent improvements and updates to your GTD app."
   - Entry list: `space-y-8`

3. **Each entry** rendered as `<article>`:
   - Wrap in a card-like container: `bg-white dark:bg-gray-900 border border-gray-200 dark:border-gray-800 rounded-xl shadow-sm p-4 tablet:p-6`
   - Date + version header row: flex with items-center gap-2
   - Date: `<time>` element with `text-sm tablet:text-base font-semibold text-gray-900 dark:text-gray-100`
   - Version badge (if present): `text-xs px-2 py-0.5 rounded-md bg-gray-100 dark:bg-gray-800 text-gray-600 dark:text-gray-400 font-medium`
   - "New" indicator: if `isNewEntry(entry.id)` is true, show a small blue dot `w-2 h-2 rounded-full bg-blue-500 animate-pulse` next to the date
   - Categories section: `space-y-3 mt-3`
   - For each category in categoryOrder, if `entry.categories[category]` exists and has items:
     - Category label row: flex with items-center gap-1.5, showing colored dot (w-1.5 h-1.5 rounded-full + getCategoryStyle dot class) and label (text-xs tablet:text-sm font-semibold uppercase tracking-wide + getCategoryStyle text class)
     - Items list: `<ul>` with `list-disc list-inside text-sm tablet:text-base text-gray-700 dark:text-gray-300 space-y-1 ml-3`

4. **Empty state** (when changelog is empty):
   - Centered text: "No updates yet." in gray

5. **Keyed each block**: Use `{#each changelog as entry (entry.id)}` for proper Svelte keying.

**Important patterns to follow:**
- Use Svelte 5 `$state` runes (NOT legacy `let` reactive declarations)
- Use `onMount` for browser-only localStorage access (SSR safety)
- Follow focus ring pattern: `focus:ring-2 focus:ring-blue-500/40 focus:ring-offset-1`
- Use app breakpoint names: `tablet:` prefix for responsive styles (NOT `md:`)
