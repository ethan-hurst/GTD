## Decisions Made

**1. Use ANALYTICS_PASSWORD env var for admin auth**
- Rationale: Reuse existing environment variable instead of requiring new FEEDBACK_PASSWORD configuration
- Impact: Zero additional setup required, follows established analytics admin pattern

**2. Store screenshots as separate blobs**
- Rationale: Avoids embedding large base64 data in item JSON, enables efficient listing without screenshot overhead
- Pattern: `items/{id}` for metadata, `screenshots/{id}` for image data
- Impact: Query performance improved for list views, screenshot only fetched on demand

**3. Honeypot spam protection**
- Rationale: Silent acceptance (not rejection) prevents bot detection of protection mechanism
- Pattern: If `botField` is truthy, return success without storing
- Impact: User-friendly anti-spam without CAPTCHA friction

**4. CORS headers on all endpoints**
- Rationale: Admin dashboard will be served from different path than API endpoints
- Pattern: Access-Control-Allow-Origin: * with OPTIONS preflight handlers
- Impact: Enables admin SPA to call API from any origin

**5. Blobs metadata as strings only**
- Rationale: Netlify Blobs metadata values must be strings (API limitation)
- Pattern: Use 'true'/'false' instead of boolean true/false for hasScreenshot
- Impact: Type coercion needed when reading metadata
