## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Added static/feedback-form.html**

- **Found during:** Task 1
- **Issue:** SvelteKit adapter-netlify generates client-side hydration shell, not server-rendered HTML with inline form content. Netlify Forms build-time crawler needs actual form HTML in static output to detect the form.
- **Fix:** Created static/feedback-form.html in addition to SvelteKit route. Static file includes identical form structure with all Netlify Forms attributes (data-netlify, honeypot, form-name).
- **Files modified:** static/feedback-form.html (created)
- **Verification:** File appears in .netlify/ build output directory with correct form HTML, data-netlify attribute present

- **Found during:** Task 1

## Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| Use static HTML file in addition to SvelteKit route | adapter-netlify creates client-side hydration shell, Netlify Forms needs static HTML at build time | Reliable form detection by Netlify's build-time crawler; SvelteKit route kept as secondary access point |
| Screenshot with html2canvas instead of FeedbackPlus | Simpler integration, no additional API learning curve, direct control over compression | Slightly more manual implementation but cleaner codebase; compression logic inline |
| JPEG compression at 0.7 quality, 0.3 if > 500KB | Balance between image quality and submission size limits | Keeps screenshots under Netlify Forms 10MB limit while maintaining readability |
| Offline TODO comment instead of full implementation | Plan 02 will add offline queue integration, no need to build incomplete solution | Clean separation of concerns between plans |
| Type selector as segmented buttons instead of dropdown | Better mobile UX, clearer visual feedback, matches modern design patterns | More visually prominent, easier to tap on mobile |

### Schema Changes

**Dexie Database Version:** 11 → 12

**New Table:**
```typescript
feedbackQueue: EntityTable<QueuedFeedback, "id">

QueuedFeedback {
  id?: number;              // Auto-increment (++id)
  type: 'bug' | 'feature';  // Feedback categorization
  description: string;      // User's feedback text
  email?: string;           // Optional contact email
  screenshot?: string;      // Optional base64 JPEG data
  timestamp: number;        // Queue time (Date.now())
  retryCount: number;       // Failed submission retry count
}
```

**Indexes:** `"++id, timestamp, retryCount"`

### Integration Points

**Netlify Forms Detection:**
- Static HTML file at `/feedback-form.html` ensures build-time detection
- Form attributes: `data-netlify="true"`, `netlify-honeypot="bot-field"`
- Hidden inputs: `<input type="hidden" name="form-name" value="feedback" />`

**Component Usage (Plan 02):**
- FeedbackModal will be triggered from Sidebar footer, mobile drawer, and 'f' keyboard shortcut
- onclose prop required: `() => void` callback
- Submission handling: online → POST to /feedback-form.html, offline → queue in Dexie

**Screenshot Capture:**
- html2canvas imported dynamically: `(await import('html2canvas')).default`
- Modal hidden during capture to exclude from screenshot
- Compression: initial 0.7 quality, further to 0.3 if > 500KB
- Base64 JPEG data stored in screenshotData state
