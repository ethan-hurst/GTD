  <name>Task 1: Create feedback-submit.mts function</name>
  <files>netlify/functions/feedback-submit.mts</files>
  <action>
    Create `netlify/functions/feedback-submit.mts` that receives POST requests with feedback data and stores it in Netlify Blobs. Follow the `analytics-track.mts` pattern exactly for function structure.

    **Request format (JSON POST):**
    ```typescript
    {
      type: 'bug' | 'feature',
      description: string,       // min 10 chars
      email?: string,
      screenshot?: string,        // base64 JPEG data
      botField?: string           // honeypot -- if non-empty, silently accept but don't store
    }
    ```

    **Implementation details:**

    1. Only allow POST method (return 405 for others)
    2. Parse JSON body with `await req.json()`
    3. Honeypot check: if `botField` is truthy (non-empty), return `{ success: true }` with 200 (silently discard -- don't store)
    4. Validate `type` is 'bug' or 'feature' (400 if not)
    5. Validate `description` is a string with length >= 10 (400 if not)
    6. Generate blob key:
       ```typescript
       const timestamp = new Date().toISOString();
       const random = Math.random().toString(16).slice(2, 6);
       const id = `${timestamp}_${random}`;
       ```
    7. Build FeedbackItem object:
       ```typescript
       const feedbackItem = {
         id,
         type,
         description,
         email: email || undefined,
         status: 'new' as const,
         hasScreenshot: !!screenshot,
         submittedAt: timestamp,
         userAgent: req.headers.get('user-agent') || 'unknown'
       };
       ```
    8. Store in `getStore('feedback')`:
       - `store.setJSON(`items/${id}`, feedbackItem, { metadata: { type, status: 'new', hasScreenshot: screenshot ? 'true' : 'false', submittedAt: timestamp } })`
       - If screenshot provided: `store.set(`screenshots/${id}`, screenshot)` (plain text, not JSON -- it's already base64)
    9. Return `{ success: true, id }` with 200
    10. Wrap in try/catch, return 500 on error

    **Important:** Blobs metadata values MUST be strings. Use `'true'`/`'false'` not `true`/`false`.

    Add CORS headers to the response: `'Access-Control-Allow-Origin': '*'` to support the admin page fetching from a different path.

    Also handle OPTIONS preflight for CORS:
    ```typescript
    if (req.method === 'OPTIONS') {
      return new Response(null, {
        status: 204,
        headers: {
          'Access-Control-Allow-Origin': '*',
          'Access-Control-Allow-Methods': 'POST, OPTIONS',
          'Access-Control-Allow-Headers': 'Content-Type'
        }
      });
    }
    ```
  </action>

  <action>
    **feedback-query.mts:**

    Create `netlify/functions/feedback-query.mts` that lists and queries feedback items with Basic Auth protection. Follow the `analytics-query.mts` pattern exactly for auth.

    1. Only allow GET (405 for others). Handle OPTIONS for CORS preflight.
    2. Auth check (copy from analytics-query.mts):
       - Check `process.env.ANALYTICS_PASSWORD` exists (503 if not)
       - Parse Authorization header for Basic Auth
       - Decode base64, split on ':', compare password
       - Return 401 with `WWW-Authenticate: Basic realm="Admin"` if invalid
    3. Parse query params from URL:
       - `status`: filter by 'new' | 'reviewed' | 'resolved' | 'archived'
       - `type`: filter by 'bug' | 'feature'
       - `id`: fetch single item (detail view)
       - `screenshot=true`: include screenshot data in detail response
    4. Single item mode (when `id` param present):
       - `const item = await store.get(`items/${id}`, { type: 'json' })`
       - If not found, return 404
       - If `screenshot=true` and `item.hasScreenshot`: `const screenshot = await store.get(`screenshots/${id}`, { type: 'text' })`
       - Return `{ item, screenshot }`
    5. List mode (no `id` param):
       - `const result = await store.list({ prefix: 'items/' })`
       - Fetch all items in parallel: `Promise.all(result.blobs.map(b => store.get(b.key, { type: 'json' })))`
       - Filter by status and type if provided
       - Sort by submittedAt descending (newest first)
       - Return `{ items, total, generatedAt }`
    6. Add CORS headers to all responses

    **feedback-update.mts:**

    Create `netlify/functions/feedback-update.mts` for updating feedback item status.

    1. Only allow PATCH (405 for others). Handle OPTIONS for CORS preflight.
    2. Same auth check as feedback-query (copy the auth block)
    3. Parse JSON body: `{ id, status, notes? }`
    4. Validate status is one of: 'new', 'reviewed', 'resolved', 'archived'
    5. Fetch existing item: `store.get(`items/${id}`, { type: 'json' })`
    6. If not found, return 404
    7. Build updated item:
       ```typescript
       const updated = {
         ...existing,
         status,
         notes: notes !== undefined ? notes : existing.notes,
         reviewedAt: new Date().toISOString()
       };
       ```
    8. Save with updated metadata:
       ```typescript
       await store.setJSON(`items/${id}`, updated, {
         metadata: {
           type: updated.type,
           status: updated.status,
           hasScreenshot: updated.hasScreenshot ? 'true' : 'false',
           submittedAt: updated.submittedAt
         }
       });
       ```
    9. Return `{ success: true, item: updated }`
    10. Add CORS headers

    **DRY tip:** The auth check is identical in both files. Since these are separate Netlify Functions (each is its own entry point), you cannot share code via a shared module in the functions directory without Netlify bundler issues. Just duplicate the auth block in both files -- it's ~20 lines and this is the established pattern (analytics-query.mts also has it inline).
