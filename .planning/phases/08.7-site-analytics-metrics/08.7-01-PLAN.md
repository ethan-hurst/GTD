  <name>Task 2: Wire PlausibleAnalytics into root layout and add offline queuing to service worker</name>
  <files>

  <action>
1. Modify `src/routes/+layout.svelte`:
   - Add import: `import PlausibleAnalytics from '@accuser/svelte-plausible-analytics';`
   - Add import: `import { PLAUSIBLE_DOMAIN, PLAUSIBLE_API_HOST } from '$lib/analytics/config';`
   - Add the `<PlausibleAnalytics domain={PLAUSIBLE_DOMAIN} apiHost={PLAUSIBLE_API_HOST} />` component right after the opening of the root template, BEFORE `<Toaster>`. This component auto-injects the Plausible script tag and handles SPA route change tracking automatically.
   - Do NOT add manual route tracking — the @accuser/svelte-plausible-analytics package handles SPA navigation automatically when placed in root layout.

2. Modify `src/service-worker.js`:
   The current service worker uses vanilla Cache API. Adding Workbox just for analytics would be heavy. Instead, implement a lightweight offline queue using IndexedDB directly in the service worker.

   Add BEFORE the existing fetch event listener:
   - Create helper functions for a simple IndexedDB-based analytics queue:
     - `openAnalyticsDB()`: Opens/creates 'analytics-queue' object store in 'analytics-offline' database
     - `queueAnalyticsRequest(url, body, headers)`: Stores failed analytics request in IndexedDB
     - `replayAnalyticsQueue()`: Replays all queued requests, deletes successful ones

   Modify the existing fetch event listener:
   - Add a check at the TOP of the fetch handler (before the `if (event.request.method !== 'GET') return;` line):
     ```
     // Intercept analytics POST requests for offline queuing
     if (event.request.method === 'POST') {
       const url = new URL(event.request.url);
       if (url.hostname === 'plausible.io' && url.pathname === '/api/event') {
         event.respondWith(handleAnalyticsRequest(event.request));
         return;
       }
     }
     ```
   - Create `handleAnalyticsRequest(request)` function that:
     - Tries to fetch normally
     - On network failure, clones request body and queues it via `queueAnalyticsRequest()`
     - Returns 202 Accepted (so Plausible client doesn't retry, we handle retries)

   Add an `online` event listener to replay queued analytics when connectivity returns:
   ```
   self.addEventListener('message', (event) => {
     if (event.data === 'replay-analytics') {
       replayAnalyticsQueue();
     }
   });
   ```

   Also trigger replay on activate event (after `self.clients.claim()`):
   ```
   replayAnalyticsQueue().catch(() => {});
   ```

   IMPORTANT: Do NOT use Workbox — the current service worker is vanilla JS and adding Workbox would require a build step change. Keep the offline queue implementation simple and self-contained using IndexedDB directly. The queue only needs to store: url, body (as text), timestamp. Retry up to 24 hours, discard older entries.

   IMPORTANT: The service worker is `.js` not `.ts`. Write plain JavaScript, no TypeScript syntax.
  </action>
