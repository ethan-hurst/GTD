  <how-to-verify>
**Pre-requisite:** You need a Plausible account with your site domain configured, and environment variables set in Netlify:
- `VITE_PLAUSIBLE_DOMAIN` = your site domain (e.g., yourapp.netlify.app)
- `PLAUSIBLE_DOMAIN` = same domain (for server-side use, future-proofing)

**After deploying to Netlify with env vars set:**

1. Visit your deployed site in a browser (not localhost)
2. Navigate to a few pages (Inbox, Actions, Calendar, Settings)
3. Open Plausible dashboard (https://plausible.io/yourdomain) — you should see:
   - Real-time visitor count (1 visitor)
   - Page visit data appearing within seconds
   - Top pages showing the routes you visited

4. To verify custom events work, you need to configure Goals in Plausible first:
   - Go to Plausible Dashboard -> Site Settings -> Goals
   - Add Custom Event goals for: `onboarding_started`, `task_created`, `task_completed`, `sync_completed`, `review_started`, `review_completed`
   - Then trigger those events in the app (add an inbox item, complete a task, etc.)
   - Check Plausible Goals section for event counts

5. To verify offline queuing:
   - Open DevTools -> Application -> Service Workers
   - Check "Offline" mode
   - Navigate to a page (this should queue an analytics event)
   - Uncheck "Offline"
   - The queued event should replay (check Network tab for plausible.io requests)

**What to look for:**
- Pageviews appearing in Plausible within ~30 seconds
- No console errors related to analytics
- Custom events appearing in Goals after configuration
