# Phase 08.7 User Setup: Plausible Analytics

This phase requires external service setup and environment configuration.

## Overview

Plausible Analytics is a privacy-respecting, cookie-free analytics platform that tracks:
- Pageviews and unique visitors
- Traffic sources and referrers
- Popular pages and navigation paths
- Custom events (after Plan 02 instrumentation)

**Why Plausible:**
- GDPR/CCPA compliant (no cookies, no personal data)
- Lightweight script (< 1KB)
- No consent banner needed
- Simple, clean dashboard

## Setup Steps

### 1. Create Plausible Account

**Where:** https://plausible.io/register

**Steps:**
1. Sign up for a Plausible account
2. Choose plan:
   - Free trial: 30 days
   - Starter plan: $9/month (10k monthly pageviews)
   - Growth plan: $19/month (100k monthly pageviews)

### 2. Add Your Site

**Where:** Plausible Dashboard → "Add a website"

**Steps:**
1. Click "+ Add a website"
2. Enter your domain:
   - Production: `yourapp.netlify.app` (or custom domain)
   - Staging: `staging--yourapp.netlify.app` (optional)
3. Select timezone
4. Click "Add snippet" (you can skip this — the app already has the integration)

### 3. Configure Environment Variables

Add these environment variables to your deployment platform (Netlify, Vercel, etc.):

| Variable | Value | Required | Description |
|----------|-------|----------|-------------|
| `VITE_PLAUSIBLE_DOMAIN` | `yourapp.netlify.app` | Yes | Your site domain (must match Plausible dashboard) |
| `PLAUSIBLE_DOMAIN` | `yourapp.netlify.app` | No | Same as above, for server-side use (future) |
| `VITE_PLAUSIBLE_API_HOST` | (leave empty) | No | Only set if using Plausible proxy |

**Example values:**
```bash
VITE_PLAUSIBLE_DOMAIN=gtd-app.netlify.app
PLAUSIBLE_DOMAIN=gtd-app.netlify.app
```

**Netlify setup:**
1. Go to Site Settings → Environment Variables
2. Add `VITE_PLAUSIBLE_DOMAIN` with your domain
3. Redeploy site for changes to take effect

**Local development:**
Create `.env` file in project root:
```bash
VITE_PLAUSIBLE_DOMAIN=localhost
```

### 4. Verify Analytics (After Deployment)

**Where:** Plausible Dashboard → Your Site

**Steps:**
1. Deploy your app with environment variables set
2. Visit your deployed site
3. Check Plausible dashboard (may take 1-2 minutes)
4. You should see:
   - Real-time visitor count
   - Pageview for your visit
   - Pages visited in "Top Pages" section

**Troubleshooting:**
- Check browser DevTools → Network tab for `plausible.io/api/event` requests
- Verify `VITE_PLAUSIBLE_DOMAIN` matches domain in Plausible dashboard exactly
- Check browser console for errors
- Ensure ad blockers aren't blocking Plausible script

### 5. Configure Custom Event Goals (After Plan 02 Deploys)

**Where:** Plausible Dashboard → Site Settings → Goals → "Add Goal" → "Custom Event"

**When:** After Plan 02 (Custom Event Instrumentation) deploys

**Events to add:**

| Goal Name | Type | Description |
|-----------|------|-------------|
| `onboarding_started` | Custom Event | User enters onboarding wizard |
| `onboarding_completed` | Custom Event | User completes onboarding |
| `task_created` | Custom Event | User creates a task |
| `task_completed` | Custom Event | User marks task complete |
| `task_deleted` | Custom Event | User deletes a task |
| `sync_initiated` | Custom Event | User starts sync |
| `sync_completed` | Custom Event | Sync finishes successfully |
| `review_started` | Custom Event | User starts weekly review |
| `review_completed` | Custom Event | User completes weekly review |
| `feature_discovered` | Custom Event | User first uses a feature |

**Steps for each event:**
1. Click "+ Add Goal"
2. Select "Custom Event"
3. Enter event name (exactly as shown above)
4. Click "Add Goal"

**Note:** These goals allow you to see custom event counts in Plausible dashboard. They're optional — events are tracked regardless, but goals make them visible in the UI.

## What Gets Tracked

### Automatic (Plan 01):
- ✓ **Pageviews** - Every route visit (/, /actions, /projects, etc.)
- ✓ **Unique visitors** - Cookie-free visitor identification
- ✓ **Traffic sources** - Where visitors come from
- ✓ **Popular pages** - Most visited routes
- ✓ **SPA navigation** - Route changes in single-page app

### Custom Events (Plan 02):
- Onboarding funnel (started, completed)
- Task lifecycle (created, completed, deleted)
- Sync usage (initiated, completed with metrics)
- Review engagement (started, completed)
- Feature discovery (first-time feature usage)

## Privacy & Compliance

**No cookies:** Plausible is cookie-free, no consent banner needed.

**No personal data:** Only aggregated, anonymous metrics collected.

**GDPR/CCPA compliant:** Plausible is built for privacy compliance.

**Data ownership:** You own your analytics data.

**Data retention:** Plausible retains data forever (unless you delete it).

**EU hosting:** Plausible uses EU servers (GDPR-compliant hosting).

## Offline Behavior

The app includes offline queuing for analytics:
- When offline, analytics events are queued in IndexedDB
- When back online, events are replayed automatically
- Events older than 24 hours are discarded
- No data loss during temporary offline periods
