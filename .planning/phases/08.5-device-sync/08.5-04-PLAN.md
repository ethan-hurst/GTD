  <name>Task 1: Implement per-record LWW merge with tombstone handling</name>
  <files>src/lib/sync/merge.ts</files>
  <action>
Create `src/lib/sync/merge.ts` with the following:

1. Import types from `$lib/db/schema` and `$lib/sync/types`.

2. Define a helper type `SyncRecord` as a generic that has at minimum `id: number` and `modified: Date` and optional `deleted?: boolean`.

3. `mergeTable<T extends SyncRecord>(local: T[], remote: T[]): T[]` - Per-record merge for a single table:
   - Create a Map keyed by record ID
   - Add all local records to the map
   - For each remote record:
     - If no local record with same ID exists: add remote record (new from other device)
     - If local record exists: compare `modified` timestamps. Use the one with later `modified` (LWW). Ensure Date comparison handles both Date objects and ISO strings (parse if string).
   - Return array from map values (do NOT filter tombstones here - tombstones must be preserved in the merged data for propagation)

4. `mergePayloads(localData: Record<string, any[]>, remoteData: Record<string, any[]>): Record<string, any[]>` - Merge entire sync payloads:
   - For each table name that exists in either local or remote:
     - If only in local: keep local
     - If only in remote: keep remote
     - If in both: call `mergeTable` with both arrays
   - Handle Date deserialization: remote data comes from JSON, so `modified`, `created`, `deletedAt`, `startTime`, `endTime`, `followUpDate`, `completedAt` fields need to be converted from ISO strings to Date objects before comparison
   - Create a `deserializeDates(record: any): any` helper that converts known date fields from string to Date

5. Export `mergeData(localData: Record<string, any[]>, remoteData: Record<string, any[]>): Record<string, any[]>` as the main entry point (calls mergePayloads).

Key considerations from research:
- Tombstones (deleted=true) are kept in merged output - they get filtered during query time, not merge time
- Clock skew is accepted as a limitation (documented, no Lamport timestamps for simplicity)
- `modified` is the single source of truth for ordering
- Date fields in JSON payloads are ISO strings and must be parsed before comparison

**Part A: Create `src/lib/sync/sync.ts`** - Sync orchestration engine:

1. Import: `encrypt`, `decrypt`, `initIVCounter` from `./crypto`; `loadPairingInfo` from `./pair`; `mergeData` from `./merge`; `db` from `$lib/db/schema`; `getSetting`, `setSetting` from `$lib/db/operations`; `SyncPayload`, `DEFAULT_SYNC_CONFIG` from `./types`.

2. `exportForSync(): Promise<SyncPayload>` - Export all database tables for sync:
   - Iterate over `db.tables`, call `table.toArray()` for each
   - Include ALL records (including deleted tombstones - do not filter)
   - Return `SyncPayload` with version=1, timestamp=ISO string, schemaVersion=db.verno, data=table data

3. `importFromSync(mergedData: Record<string, any[]>): Promise<void>` - Import merged data into IndexedDB:
   - Use a Dexie transaction for atomicity
   - For each table: clear existing data, bulkPut merged records
   - This is a full-replace with the merged result (not incremental)

4. `pullFromCloud(deviceId: string): Promise<string | null>` - Fetch encrypted blob from Netlify:
   - `fetch('/.netlify/functions/sync-pull?deviceId=' + deviceId)`
   - Parse response JSON. If `found === false`, return null
   - Return `encryptedBlob` string

5. `pushToCloud(deviceId: string, encryptedBlob: string): Promise<boolean>` - Push encrypted blob to Netlify:
   - `fetch('/.netlify/functions/sync-push', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ deviceId, encryptedBlob }) })`
   - Return true if response.ok

6. `performSync(pairingCode: string, deviceId: string): Promise<{ success: boolean; error?: string }>` - Full sync cycle:
   - Step 1: Pull from cloud
   - Step 2: If remote data exists, decrypt it
   - Step 3: Export local data
   - Step 4: If remote data exists, merge local + remote; otherwise use local only
   - Step 5: Encrypt merged (or local) data
   - Step 6: Push to cloud
   - Step 7: If merge happened, import merged data into local DB
   - Step 8: Save lastSyncTime to settings
   - Wrap all steps in try/catch, return `{ success, error }`

7. `SyncDebouncer` class - Debounced sync triggering:
   - `private changeCount = 0`
   - `private timeoutId: ReturnType<typeof setTimeout> | null = null`
   - `queueSync(syncFn: () => Promise<void>)`: Increment changeCount. If >= MAX_CHANGES (5), execute immediately. Otherwise, set/reset timeout for MAX_DELAY_MS (2000).
   - `cancel()`: Clear timeout, reset counter
   - Export singleton `syncDebouncer`
