## Decisions Made

**1. Device ID as primary key**
- Device ID is SHA-256 hash of pairing code (64-char hex)
- Sent from client, used as blob store key
- Format validation (regex check) prevents abuse

**2. Security through encryption**
- No authentication layer - knowing device ID is sufficient
- Server never sees plaintext data
- All encryption/decryption happens client-side
- Server stores opaque encrypted strings

**3. Payload limits**
- 10 MB max payload size enforced in sync-push
- Typical GTD data << 1 MB, 10 MB is safety valve
- Returns 413 Payload Too Large if exceeded

**4. Error response patterns**
- sync-pull returns `{ found: false }` (200) when no data exists
- Not using 404 - helps client distinguish "no sync data" from actual errors
- Consistent JSON responses with Content-Type headers across all paths

**5. Netlify Blobs integration**
- Uses `getStore('sync-data')` for persistent key-value storage
- Zero-config - no database setup required
- `{ type: 'text' }` option retrieves as string (not ReadableStream)
