**Test:** 
1. Open app in Browser A, go to Settings, click "Generate Pairing Code"

**Why human:** Requires two browser instances/devices, network calls to Netlify Functions

#### 2. Two-Way Sync Verification

**Test:**
1. After pairing (from Test 1), create an inbox item in Browser A: "Test from Device A"
2. Wait 2 seconds (debounce delay)
3. In Browser B, refresh page or wait for visibility change sync

**Expected:**

- No duplicate items
- Both devices show same last sync time

**Why human:** Requires observing real-time sync behavior across devices

#### 3. Conflict Resolution (Last-Write-Wins)

**Test:**
1. After pairing, disconnect Browser A from network (dev tools offline mode)
2. In Browser A, create item "Offline Item A"
3. In Browser B (still online), create item "Online Item B"
4. Reconnect Browser A and wait for sync

**Expected:**
- Both items appear on both devices (new items preserved)
- If same item edited on both, newer version wins based on modified timestamp
- No data loss

**Why human:** Requires simulating offline scenarios and observing merge behavior

#### 4. Deletion Propagation via Tombstones

**Test:**
1. After pairing, create item "Delete Test" on Browser A, wait for sync
2. Verify item appears on Browser B
3. Delete "Delete Test" on Browser A, wait for sync

**Expected:**
- Item disappears from Browser B's UI
- Item still exists in IndexedDB with `deleted=true` (check via dev tools)
- Tombstone propagates deletion across devices

**Why human:** Requires inspecting IndexedDB and observing deletion sync

#### 5. Encryption Verification

**Test:**
1. After pairing and syncing data, open Network tab in dev tools
2. Monitor sync-push POST request payload

**Expected:**
- Payload `encryptedBlob` field contains base64-encoded JSON
- JSON has structure `{salt, iv, ciphertext}` (all base64 strings)
- No plaintext GTD data visible in network payload
- Wrong pairing code on second device fails to decrypt

**Why human:** Requires network inspection and attempting wrong pairing code

#### 6. Auto-Sync on App Open

**Test:**
1. After pairing, create item on Browser A, wait for sync
2. Close Browser B completely
3. Create another item on Browser A, wait for sync
4. Reopen Browser B

**Expected:**
- Browser B automatically syncs on open (no manual "Sync Now" needed)
- New item from Browser A appears immediately
- Sync status shows "Syncing..." briefly then "Synced X seconds ago"

**Why human:** Requires observing lifecycle behavior across browser sessions

#### 7. Unpair and Re-pair

**Test:**

**Expected:**
- After unpair, sync status disappears from UI
