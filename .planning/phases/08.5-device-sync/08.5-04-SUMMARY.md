## Decisions Made

1. **Last-Write-Wins uses modified timestamp**: Single source of truth for conflict resolution. Simpler than Lamport clocks, clock skew accepted as documented limitation.

2. **Tombstones preserved in merge output**: Deleted records with `deleted=true` kept in merged data. Filtering happens at query time (existing operations.ts), not merge time. Ensures deletions propagate across devices.

3. **Date deserialization from JSON**: Remote payloads come from JSON, so all date fields (created, modified, deletedAt, startTime, endTime, followUpDate, completedAt) converted from ISO strings to Date objects before comparison.

4. **Full-replace import strategy**: After merge, clear existing table data and bulkPut merged records. Simpler than incremental updates, ensures consistency, acceptable for current data size.

5. **Pairing code in memory only**: Never persisted to IndexedDB. After page refresh, user sees "re-enter pairing code" prompt. Balances security and UX simplicity.

6. **Debouncer triggers after 2000ms OR 5 changes**: Whichever comes first. Prevents excessive API calls while ensuring timely sync.
