# Phase 08.5 Plan 02: Client-Side Encryption & Pairing Summary

**One-liner:** AES-GCM encryption with PBKDF2 key derivation and 6-character unambiguous pairing codes using Web Crypto API.

## What Was Built

### Crypto Module (`src/lib/sync/crypto.ts`)

Implemented client-side encryption foundation:

1. **Key Derivation:**
   - `deriveKey()` - PBKDF2 with 100,000 iterations using SHA-256
   - Converts pairing code + salt into AES-GCM 256-bit key
   - Non-extractable keys (never leave Web Crypto API)

2. **Salt Generation:**
   - `generateSalt()` - 16-byte random salt using crypto.getRandomValues()
   - Fresh salt for each encryption operation

3. **Counter-Based IV Generation:**
   - `initIVCounter()` - Initialize counter from IndexedDB with device-specific prefix
   - `getNextIV()` - Generate 12-byte IV: [4 bytes device prefix][8 bytes counter]
   - Counter persisted to IndexedDB after each increment
   - Device prefix from SHA-256 hash of deviceId prevents cross-device IV collision

4. **Encryption/Decryption:**
   - `encrypt()` - Full encryption flow: generate salt, derive key, get IV, encrypt with AES-GCM
   - `decrypt()` - Full decryption flow: parse blob, derive key, decrypt
   - Returns/accepts JSON blob: `{ salt: base64, iv: base64, ciphertext: base64 }`
   - Descriptive error messages (e.g., "Decryption failed - wrong pairing code?")

### Pairing Module (`src/lib/sync/pair.ts`)

Implemented pairing code lifecycle:

1. **Code Generation:**
   - `generatePairingCode()` - Random 6-character code from unambiguous charset
   - Character set: `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (32 chars)
   - Excludes ambiguous: 0/O, 1/I/L
   - Returns formatted: `XXX-XXX`

2. **Validation & Normalization:**
   - `validatePairingCode()` - Check format and character validity
   - `normalizePairingCode()` - Strip dashes/whitespace, uppercase, validate length
   - `formatPairingCode()` - Format as `XXX-XXX` for display

3. **Hashing & Storage:**
   - `hashPairingCode()` - SHA-256 hash to 64-char hex deviceId
   - `savePairingInfo()` - Store { deviceId, pairedAt } in IndexedDB (not raw code)
   - `loadPairingInfo()` - Retrieve pairing info
   - `clearPairingInfo()` - Unpair device

## Technical Implementation

### Web Crypto API Usage

All cryptographic operations use browser-native `crypto.subtle` API:

- **PBKDF2:** `crypto.subtle.importKey()` + `crypto.subtle.deriveKey()`
- **AES-GCM:** `crypto.subtle.encrypt()` / `crypto.subtle.decrypt()`
- **SHA-256:** `crypto.subtle.digest()` for pairing code hashing
- **Random:** `crypto.getRandomValues()` for salt and pairing code generation

### IV Counter Architecture

Counter-based IV generation with device-specific prefixes:

```
IV = [4-byte device prefix][8-byte counter]
     └─ SHA-256(deviceId)[0:4]
                          └─ BigInt, incremented sequentially
```

**Benefits:**
- Guaranteed uniqueness per device (device prefix)
- No cross-device IV collision (even with same counter value)
- No random IV collision risk (sequential, not random)
- Counter persisted to IndexedDB for recovery after page reload

### Pairing Code Design

**Format:** 6 characters, unambiguous alphanumeric
**Example:** `ABC-234` (3+3 with dash for display)

**Character Set Rationale:**
- 32 characters (ABCDEFGHJKLMNPQRSTUVWXYZ23456789)
- Excludes: 0 (looks like O), 1 (looks like I/L)
- Entropy: log2(32^6) ≈ 30 bits (sufficient for short-lived codes)

**Security Model:**
- Raw pairing code only in memory during pairing flow
- SHA-256 hash stored as deviceId in IndexedDB
- Hash used as blob storage key in Netlify Functions
- Pairing code doubles as encryption key (via PBKDF2)

## Testing Notes

**Manual Testing Required:**

1. **Encryption Round-Trip:**
   ```javascript
   import { encrypt, decrypt, initIVCounter } from '$lib/sync/crypto';
   await initIVCounter('test-device');
   const encrypted = await encrypt('test data', 'ABC234');
   const decrypted = await decrypt(encrypted, 'ABC234');
   console.assert(decrypted === 'test data');
   ```

2. **Pairing Code Format:**
   ```javascript
   import { generatePairingCode, validatePairingCode } from '$lib/sync/pair';
   const code = generatePairingCode(); // e.g., "ABC-234"
   console.assert(validatePairingCode(code));
   console.assert(validatePairingCode('ABC234')); // without dash
   console.assert(!validatePairingCode('ABC-2I4')); // invalid char 'I'
   ```

   - Encrypt data, reload page, encrypt again
