<task type="auto">
  <name>Task 1: Implement crypto module with AES-GCM and PBKDF2</name>
  <files>src/lib/sync/crypto.ts</files>
  <action>
Create `src/lib/sync/crypto.ts` with the following exports:

1. `generateSalt(): Uint8Array` - Generate 16-byte random salt using `crypto.getRandomValues()`.

2. `deriveKey(pairingCode: string, salt: Uint8Array): Promise<CryptoKey>` - Derive AES-GCM 256-bit key from pairing code using PBKDF2:
   - Import pairing code as raw key material via `crypto.subtle.importKey('raw', ...)`
   - Use `crypto.subtle.deriveKey` with PBKDF2 params: salt, 100000 iterations, SHA-256
   - Output: AES-GCM key, 256-bit, not extractable, usages: ['encrypt', 'decrypt']

3. `getNextIV(): Uint8Array` - Counter-based IV generation:
   - Maintain an internal counter (BigInt) initialized from IndexedDB on first call
   - Increment counter, persist to IndexedDB settings (key: 'sync-iv-counter')
   - Convert counter to 12-byte Uint8Array (96-bit IV) using DataView
   - To avoid cross-device IV collision, use device-specific prefix: first 4 bytes from hashed device ID, last 8 bytes from counter
   - Export `initIVCounter(deviceId: string): Promise<void>` to initialize the counter with device prefix

4. `encrypt(data: string, pairingCode: string): Promise<string>` - Full encryption flow:
   - Generate fresh salt (16 bytes)
   - Derive key from pairing code + salt
   - Get next IV from counter
   - Encrypt data using `crypto.subtle.encrypt({ name: 'AES-GCM', iv }, key, encodedData)`
   - Return JSON string with structure: `{ salt: base64, iv: base64, ciphertext: base64 }`
   - Use helper functions for Uint8Array <-> base64 conversion (avoid deprecated btoa/atob for binary):
     ```typescript
     function toBase64(bytes: Uint8Array): string {
       let binary = '';
       for (const byte of bytes) binary += String.fromCharCode(byte);
       return btoa(binary);
     }
     function fromBase64(b64: string): Uint8Array {
       const binary = atob(b64);
       const bytes = new Uint8Array(binary.length);
       for (let i = 0; i < binary.length; i++) bytes[i] = binary.charCodeAt(i);
       return bytes;
     }
     ```

5. `decrypt(encryptedBlob: string, pairingCode: string): Promise<string>` - Full decryption flow:
   - Parse JSON to extract salt, iv, ciphertext (all base64)
   - Decode base64 to Uint8Array
   - Derive key from pairing code + salt
   - Decrypt using `crypto.subtle.decrypt({ name: 'AES-GCM', iv }, key, ciphertext)`
   - Return decoded plaintext string

All functions must use the Web Crypto API (crypto.subtle). No external dependencies. Handle errors with descriptive messages (e.g., "Decryption failed - wrong pairing code?").

  <done>crypto.ts exports deriveKey, encrypt, decrypt, generateSalt, getNextIV, initIVCounter. Uses PBKDF2 with 100k iterations, AES-GCM 256-bit, counter-based 12-byte IV.</done>
</task>

<task type="auto">
  <name>Task 2: Implement pairing code generation and validation</name>
  <files>src/lib/sync/pair.ts</files>
  <action>
Create `src/lib/sync/pair.ts` with the following exports:

1. `generatePairingCode(): string` - Generate 6-character pairing code:
   - Use character set: `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (no ambiguous: 0/O, 1/I/L)
   - Use `crypto.getRandomValues(new Uint8Array(6))` for random values
   - Map each random byte to character set using modulo
   - Return formatted as 3+3 with dash: `XXX-XXX` (for display), store without dash

2. `validatePairingCode(code: string): boolean` - Validate pairing code format:
   - Strip dashes and whitespace, convert to uppercase
   - Must be exactly 6 characters
   - All characters must be in the allowed character set
   - Return true if valid

3. `normalizePairingCode(code: string): string` - Normalize user input:
   - Strip dashes, whitespace, convert to uppercase
   - Return clean 6-character string (or throw if invalid)

4. `formatPairingCode(code: string): string` - Format for display:
   - Takes 6-char code, returns `XXX-XXX` format

5. `hashPairingCode(pairingCode: string): Promise<string>` - Hash pairing code for use as blob storage key (device ID):
   - Use `crypto.subtle.digest('SHA-256', encodedCode)`
   - Return hex string (64 chars)
   - This hash is what gets sent to Netlify Functions as `deviceId`

6. `savePairingInfo(pairingCode: string): Promise<void>` - Store pairing info in IndexedDB:
   - Hash the pairing code for deviceId
   - Store in settings table: key='sync-pairing', value={ deviceId: hashedCode, pairedAt: new Date().toISOString() }
   - Do NOT store the raw pairing code (security: avoid plaintext exposure in IndexedDB)

7. `loadPairingInfo(): Promise<{ deviceId: string; pairedAt: Date } | null>` - Load pairing info from IndexedDB settings

8. `clearPairingInfo(): Promise<void>` - Remove pairing info from settings (unpair)

Import `getSetting` and `setSetting` from `$lib/db/operations` for IndexedDB storage. Import from `$lib/db/schema` if needed for direct db access.
  </action>
