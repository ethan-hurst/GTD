Create `src/lib/components/OnboardingWizard.svelte` — a full-screen overlay wizard with 5 steps. This is an overlay that covers the entire viewport (z-50, fixed position, bg-white dark:bg-gray-950).

**Structure:**
- Full-screen fixed overlay (not a modal — fills the screen with centered content)
- Progress indicator at top showing step X of 5 with progress bar (same pattern as WeeklyReviewWizard)
- Central content area (max-w-xl mx-auto, centered vertically) with step content
- Navigation buttons at bottom: Back / Next (or Skip on welcome, Finish on last step)

**Step 1 — Welcome:**
- Heading: "Welcome to GTD"
- Brief explanation: "Getting Things Done is a methodology for managing everything on your plate. We'll walk through the 4 key concepts in about 60 seconds."
- Two buttons: "I know GTD — skip" (calls onboardingState.skipOnboarding()) and "Let's go" (calls onboardingState.next())
- Skip button styled as text/secondary, "Let's go" as primary
- Skip option ONLY appears on this first step per CONTEXT.md requirement

**Step 2 — Capture:**
- Heading: "Step 1: Capture Everything"
- Explanation: "GTD starts with getting everything out of your head. Anything on your mind — tasks, ideas, commitments — goes into your Inbox."
- Embedded REAL input field (NOT InboxCapture component — create a simplified inline version that uses addItem directly). Placeholder: "Try it — type something on your mind..."
- Suggested prompts below input as gray text: "e.g., Call dentist, Plan weekend trip, Review budget"
- On submit: call addItem({title, type:'inbox', notes:''}), fire confetti (particleCount:50, spread:60, origin:{y:0.6}, disableForReducedMotion:true), show brief "Captured!" text, call onboardingState.completeStep('capture')
- User must capture at least one item before Next button enables. Show Next button disabled until first capture.
- After capture + confetti, enable Next with brief delay (1s)

**Step 3 — Process:**
- Heading: "Step 2: Process to Zero"
- Explanation: "Once captured, process each item: Is it actionable? Will it take less than 2 minutes? Can you delegate it? The Processing Flow guides you through these decisions."
- Show a visual representation of the GTD decision tree (not interactive — just explanatory):
  - "Is it actionable?" -> No: Trash / Someday/Maybe / Reference
  - "Is it actionable?" -> Yes: "Less than 2 min?" -> Do it now
  - "Can someone else do it?" -> Delegate (Waiting For)
  - "Otherwise" -> Next Action (assign a context like @computer, @phone)
- Style as a simple flowchart using Tailwind (flex/grid layout with arrows/connectors, not SVG)
- Next button always enabled (informational step)

**Step 4 — Organize:**
- Heading: "Step 3: Organize by Lists"
- Explanation: "GTD organizes your work into clear lists so you always know what to do next:"
- Show 4 cards/tiles in a 2x2 grid:
  - "Next Actions" — "Tasks you can do right now, filtered by context (@computer, @phone, @office)"
  - "Projects" — "Outcomes requiring multiple steps. Every project needs at least one next action."
  - "Waiting For" — "Items you've delegated or are waiting on someone else for."
  - "Someday/Maybe" — "Ideas and possibilities for the future. Not now, but not forgotten."
- Each card has a small icon or emoji and brief description
- Next button always enabled

**Step 5 — Review Intro:**
- Heading: "Step 4: Review Weekly"
- Explanation: "The Weekly Review is the secret to making GTD work. Once a week, review everything: empty your inbox, check all projects, follow up on delegations."
- Brief note: "We'll remind you when it's time for your weekly review."
- Primary "Get Started" button (calls onboardingState.finishOnboarding())
- On finish: brief confetti celebration (particleCount:100, spread:70), then wizard closes

**Technical details:**
- Import onboardingState from '$lib/stores/onboarding.svelte.ts'
- Import confetti from 'canvas-confetti'
- Import addItem from '$lib/db/operations'
- Import inboxState from '$lib/stores/inbox.svelte' (to reload after capture)
- Use Svelte 5 $state runes for local UI state (capturedFirst, isSubmitting)
- Step transitions: simple opacity/transform transition using Svelte transition:fade or CSS transitions
- All text content inline in the component (no separate content files)
- Dark mode support on all elements (dark: classes)
- Responsive: works on mobile (single column, smaller padding)
