<task type="auto">
  <name>Task 1: Wire OnboardingWizard and route tracking into root layout</name>
  <files>src/routes/+layout.svelte</files>
  <action>
Modify `src/routes/+layout.svelte` to add:

1. **Onboarding wizard overlay:**
   - Import OnboardingWizard from '$lib/components/OnboardingWizard.svelte'
   - Import onboardingState from '$lib/stores/onboarding.svelte'
   - In onMount, call `await onboardingState.loadState()` — this loads persisted state from settings table
   - After loadState, call `const shouldShow = await onboardingState.shouldShowOnboarding()` — if true, call `onboardingState.startOnboarding()`
   - Conditionally render: `{#if onboardingState.isActive}<OnboardingWizard />{/if}` — place this AFTER the main app layout div so it overlays on top (z-50)
   - The wizard is a fixed overlay, so the app layout renders behind it but is not interactive while wizard is active

2. **Route-based feature tracking:**
   - Import { getFeatureFromRoute, markFeatureVisited } from '$lib/utils/featureTracking'
   - Import { page } from '$app/stores'
   - Add a `let previousPath = $state('')` variable
   - Add $effect that watches `$page.url.pathname`:
     ```
     $effect(() => {
       const currentPath = $page.url.pathname;
       if (currentPath !== previousPath && (onboardingState.hasCompleted || onboardingState.hasSkipped)) {
         previousPath = currentPath;
         const feature = getFeatureFromRoute(currentPath);
         if (feature) {
           markFeatureVisited(feature).catch(() => {});
         }
       }
     });
     ```
   - Only track AFTER onboarding is completed/skipped (don't track during wizard)

3. **Manual tracking for search and keyboard shortcuts:**
   - In the existing handleKeydown function, add tracking for keyboard shortcuts:
     - When ANY single-key shortcut (n/p/w/s/r//) is used, call `markFeatureVisited('keyboard-shortcuts')` (in addition to existing navigation)
   - In the search focus handler (Cmd+K), call `markFeatureVisited('search')`
   - These calls should be fire-and-forget (no await, .catch(() => {}))

**Preserve ALL existing layout functionality:**
- Keep Sidebar, SearchBar, StatusBar, Toaster exactly as they are
- Keep ALL existing keyboard shortcuts unchanged
- Keep theme listener, storage status request
- Only ADD the onboarding overlay, route tracking, and manual feature tracking

**What NOT to do:**
- Do NOT change the app structure or layout
- Do NOT make onboarding blocking (app loads normally, wizard overlays)
- Do NOT load onboarding state synchronously — use onMount async pattern
