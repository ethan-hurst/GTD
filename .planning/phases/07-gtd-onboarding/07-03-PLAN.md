Export function `getHintContent(feature: Feature, hasSkipped: boolean): HintContent` — returns from appropriate set.

Create `src/lib/components/FeatureHint.svelte`:
- Props: `feature: Feature` (which feature this hint is for), `children` (snippet), `position?: 'top' | 'bottom' | 'left' | 'right'` (default 'right')
- On mount: check `hasVisitedFeature(feature)` — if visited, don't show hint
- If NOT visited, render a small tooltip indicator (pulsing blue dot or subtle glow) next to the wrapped element
- Use svelte-pops Popover component to show the hint content on hover/click of the indicator
- Hint content: bold title + description from getHintContent(), using onboardingState.hasSkipped for content selection
- The hint does NOT have a dismiss button — it stays until the feature is actually used (tracked by featureTracking.ts)
- Once hasVisitedFeature returns true (checked reactively or on route change), the hint indicator disappears
- Accessibility: use aria-describedby linking the hint to its target element
- Escape key closes popover (handled by svelte-pops by default)
- Dark mode support on hint popover styling

**Svelte-Pops usage pattern:**
```svelte
<script>
  import { Popover } from 'svelte-pops';
</script>
```
Use the Popover component with appropriate placement prop. If svelte-pops API differs from research (check actual exports after install), adapt to the actual API — the key requirement is a positioned popover, not a specific API shape.

**IMPORTANT:** If svelte-pops does not work as expected after install (wrong exports, incompatible API), fall back to Svelte 5 native approach: use a simple absolute-positioned div with manual positioning via a wrapper div with `position: relative`. This is acceptable for sidebar nav hints which have predictable positions. Do NOT spend time debugging library issues — fall back quickly.

**What NOT to do:**
- Do NOT make hints dismissible via a close button — they disappear only on feature use
- Do NOT show hints during the onboarding wizard (wizard handles its own education)
- Do NOT show hints if onboarding hasn't been completed or skipped yet (check onboardingState.hasCompleted || onboardingState.hasSkipped)
