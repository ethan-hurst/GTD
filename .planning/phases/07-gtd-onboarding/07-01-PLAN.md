  <files>src/lib/stores/onboarding.svelte.ts, src/lib/utils/featureTracking.ts</files>
  <action>
Create `src/lib/stores/onboarding.svelte.ts` following the WeeklyReviewState pattern from `src/lib/stores/review.svelte.ts`:

OnboardingState class with Svelte 5 $state runes:
- `currentStep = $state<OnboardingStep>('welcome')` where OnboardingStep = 'welcome' | 'capture' | 'process' | 'organize' | 'review-intro'
- `completedSteps = $state<Set<OnboardingStep>>(new Set())`
- `isActive = $state(false)` — controls whether wizard overlay is showing
- `hasSkipped = $state(false)` — tracks if user chose "I know GTD"
- `hasCompleted = $state(false)` — tracks if user finished full walkthrough
- `stepOrder` readonly array of the 5 steps
- `stepLabels` and `stepDescriptions` records for each step (GTD-themed: Welcome to GTD, Capture, Process, Organize, Review)
- $derived: `currentStepIndex`, `progress` (percentage), `canGoBack`, `canGoNext`, `isComplete`
- Methods:
  - `async loadState()` — reads onboardingCompleted, onboardingSkipped, onboardingStep from settings. Includes Safari eviction fallback: if no flags but db.items.count() > 0, treat as completed.
  - `startOnboarding()` — sets isActive=true, resets steps
  - `completeStep(step)` — adds to completedSteps (new Set pattern for reactivity), persists current step to settings
  - `next()` / `back()` — step navigation
  - `async skipOnboarding()` — sets onboardingSkipped=true in settings, sets hasSkipped=true, isActive=false
  - `async finishOnboarding()` — sets onboardingCompleted=ISO string in settings, isActive=false, hasCompleted=true
  - `async shouldShowOnboarding(): Promise<boolean>` — returns true if not completed AND not skipped AND no items in DB (Safari fallback)
  - `async resetOnboarding()` — clears onboardingCompleted, onboardingSkipped, onboardingStep from settings, clears all feature_visited_ keys. Used by Settings page "Show onboarding again" toggle.
- Export singleton: `export const onboardingState = new OnboardingState()`

Create `src/lib/utils/featureTracking.ts`:
- Export type `Feature = 'inbox' | 'next-actions' | 'projects' | 'waiting' | 'someday' | 'review' | 'search' | 'keyboard-shortcuts' | 'settings'`
- `routeToFeature` record mapping paths to Feature: '/' -> 'inbox', '/actions' -> 'next-actions', '/projects' -> 'projects', '/waiting' -> 'waiting', '/someday' -> 'someday', '/review' -> 'review', '/settings' -> 'settings'
- `async markFeatureVisited(feature: Feature)` — calls setSetting(`feature_visited_${feature}`, true)
- `async hasVisitedFeature(feature: Feature): Promise<boolean>` — calls getSetting, returns boolean
- `getFeatureFromRoute(path: string): Feature | null` — looks up in routeToFeature map
- `async getAllVisitedFeatures(): Promise<Set<Feature>>` — loads all feature_visited_ keys, returns Set of visited features
- `async clearAllFeatureVisits(): Promise<void>` — clears all feature_visited_ keys from settings (used by reset)

Use getSetting/setSetting from '$lib/db/operations' for all persistence. Do NOT create new database tables — use existing settings table with &key constraint.
