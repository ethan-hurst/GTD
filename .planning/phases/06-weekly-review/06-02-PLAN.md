  <name>Task 1: WeeklyReviewWizard component</name>
  <files>src/lib/components/WeeklyReviewWizard.svelte</files>
  <action>
Create WeeklyReviewWizard.svelte — a full-page wizard component for the weekly review.

**Layout structure (top to bottom):**
1. **Progress bar** at top — Tailwind horizontal bar showing `weeklyReviewState.progress`% filled. Text below: "{N} of 8 steps complete ({X}%)".
2. **Step sidebar list** on the left — vertical list of all 8 steps. Each step shows:
   - Step number (1-8)
   - Step label from `weeklyReviewState.stepLabels`
   - Visual state: completed (green checkmark), current (blue highlight/bold), pending (gray)
   - Clickable — calls `weeklyReviewState.goToStep(step)` to navigate
3. **Step content** on the right — shows current step's detail:
   - Step title (large, from stepLabels)
   - Step description (from stepDescriptions)
   - Item count context (see below)
   - "Mark Step Complete" button — calls `weeklyReviewState.completeStep(currentStep)` then auto-advances to next incomplete step
4. **Navigation footer** — "Back" button (disabled if first step), "Next" button (disabled if last step), "Finish Review" button (visible only when `weeklyReviewState.isComplete`)

**Item count context for each step** (import existing state stores):
- `inbox`: Show `inboxState.itemCount` items ("X items to process")
- `calendar-past`: "Review your past calendar for follow-ups" (no count — calendar not yet integrated)
- `calendar-future`: "Review upcoming events" (no count — calendar not yet integrated)
- `actions`: Show count from actionState ("X next actions to review")
- `waiting`: Show count from waitingForState ("X items waiting on others")
- `projects`: Show count from projectState, include stalled count ("X projects, Y stalled")
- `someday`: Show count from somedayMaybeState ("X someday/maybe items")
- `creative`: "Capture any new ideas that came to mind during this review" (no count)

**For steps with zero items** (empty states): Show encouraging message like "No items here — looking good!" in green text.

**Styling:** Follow existing Tailwind patterns. Use the established dark mode classes (dark:bg-gray-*, dark:text-gray-*). The step sidebar should be ~240px wide (matching app sidebar width pattern). Use transition-all duration-300 on the progress bar fill.

**Responsive layout:** Use flex layout. Step sidebar on left, step content flex-1 on right. On the step content side, center the content with max-w-2xl mx-auto padding.

**Important implementation notes:**
- Import weeklyReviewState from '$lib/stores/review.svelte'
- Import inboxState, actionState, projectState, waitingForState, somedayMaybeState from their respective stores
- Load all state store items on mount (call loadItems/loadProjects on each)
- When marking a step complete, auto-advance: after completeStep(), find the next non-completed step and goToStep() to it. If all complete, stay on current step.
- The "Finish Review" button should NOT be in this component — that logic goes in Plan 06-03 (integration with confetti). For now, emit an `onfinish` callback prop.
  </action>
