  <name>Task 1: Create WaitingForItem component</name>
  <files>src/lib/components/WaitingForItem.svelte</files>
  <action>
  Create WaitingForItem.svelte following the ActionItem.svelte and ProjectItem.svelte patterns.

  Props interface:
  - `item: GTDItem` (the waiting-for item)
  - `isOverdue: boolean` (controls visual highlight)
  - `isExpanded: boolean` (controls detail panel visibility)
  - `onToggleExpand: () => void` (callback for expand/collapse)
  - `onResolved: () => void` (callback to reload list after resolve)

  Component structure:
  1. **Main row** — clickable to expand/collapse. Contains:
     - Title text (bold)
     - "Delegated to: {item.delegatedTo}" subtitle (text-sm, gray)
     - "Delegated: {formatDate(item.created)}" (text-xs, gray)
     - If followUpDate exists: "Follow up: {formatDate(item.followUpDate)}" (text-xs; red if overdue, gray otherwise)
     - Resolve button (checkmark icon or "Resolve" text) on the right side

  2. **Overdue styling** — When isOverdue is true:
     - Container: `bg-red-50 dark:bg-red-900/20 border border-red-200 dark:border-red-800`
     - Follow-up date text: `text-red-700 dark:text-red-300`
     - When NOT overdue: standard `bg-white dark:bg-gray-800` styling

  3. **Expand panel** — When isExpanded, show below the main row:
     - Notes field (editable textarea, auto-saves on blur using updateItem)
     - Follow-up date field (`<input type="date">`, auto-saves on change using updateItem)
     - Person field (editable input, auto-saves on blur using updateItem)
     - Project link display (if projectId set, show project name -- read-only for now)
     - Delete button (with confirmation or just trash icon)

  4. **Resolve action** — When resolve button clicked:
     - Call `resolveWaitingFor(item.id)` from operations
     - Show toast: "Waiting-for item resolved" with undo action
     - Undo calls the undo function returned by resolveWaitingFor
     - Call onResolved() to refresh the list
     - Use the same toast undo pattern from ActionItem (svelte-5-french-toast)

  Date formatting: Use `new Date(date).toLocaleDateString()` for display. For relative time on delegation date, use the existing relative time formatter if available, or simple toLocaleDateString.

  Use native `<input type="date">` for follow-up date editing (NOT a custom date picker).
  </action>
