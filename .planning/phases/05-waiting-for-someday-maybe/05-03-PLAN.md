  <action>
  Create SomedayMaybeItem.svelte following the ProjectItem/ActionItem patterns.

  Props interface:
  - `item: GTDItem` (the someday/maybe item)
  - `isExpanded: boolean` (controls detail panel visibility)
  - `onToggleExpand: () => void` (callback for expand/collapse)
  - `onChanged: () => void` (callback to reload list after promote/delete/update)

  Component structure:
  1. **Main row** -- clickable to expand/collapse. Contains:
     - Title text (bold)
     - Category badge (if item.category is set): small pill badge showing category name, muted styling (bg-gray-100 dark:bg-gray-700, text-gray-600 dark:text-gray-300, text-xs, rounded-full, px-2 py-0.5)
     - "Added: {formatDate(item.created)}" (text-xs, gray)
     - Promote button on the right side (dropdown or two-button group)

  2. **Promote action** -- Button that reveals a dropdown or two adjacent buttons:
     - "Make Project" -- calls promoteSomedayToActive(item.id, 'project'), toast "Promoted to Projects", call onChanged()
     - "Make Action" -- calls promoteSomedayToActive(item.id, 'next-action'), toast "Promoted to Next Actions", call onChanged()
     - Implementation: Use a simple button group (two small buttons side by side) OR a dropdown menu. Prefer button group for simplicity -- two buttons labeled "Project" and "Action" with an upward arrow/promote icon.
     - Keep it simple -- no modal, no multi-step wizard. Research confirms promotion is just a type change.

  3. **Expand panel** -- When isExpanded, show below the main row:
     - Notes field (editable textarea, auto-saves on blur using updateItem)
     - Category selector: row of predefined category buttons (import SOMEDAY_CATEGORIES from store). Click to assign, click again to unassign. Auto-save via updateItem on change.
     - Delete button (trash icon, calls deleteItem, toast "Item deleted", onChanged())

  Use updateItem from operations.ts for all inline edits. Toast notifications via svelte-5-french-toast.
