<task type="auto">
  <name>Task 1: Polish OnboardingWizard</name>
  <files>src/lib/components/OnboardingWizard.svelte</files>
  <action>
Read OnboardingWizard.svelte first to understand the full structure.

Polish OnboardingWizard.svelte applying the same patterns established in Plans 01-06:

1. **Overlay/backdrop:**
   - If using a fixed overlay, ensure `backdrop-blur-sm bg-black/50` for modern feel
   - Modal container: ensure `shadow-2xl rounded-xl` for premium elevation

2. **Step indicators/dots (if present):**
   - Ensure active dot uses blue-600 with `transition-all duration-200`
   - Completed dots use green-600 or blue-600

3. **Headings:**
   - Step titles: `text-2xl font-bold text-gray-900 dark:text-gray-100 tracking-tight`
   - Step descriptions: `text-base text-gray-600 dark:text-gray-300`

4. **All buttons:**
   - Primary (Next/Get Started): add `focus:outline-none focus:ring-2 focus:ring-blue-500/40 focus:ring-offset-2`, `active:scale-[0.98]`, `shadow-sm hover:shadow-md`, `transition-all duration-150`
   - Secondary (Back/Skip): add `focus:outline-none focus:ring-2 focus:ring-gray-400/40 focus:ring-offset-2`, `transition-all duration-150`
   - Standardize sizing: primary buttons `px-6 py-3 text-base rounded-lg` (hero CTA size for onboarding), secondary `px-4 py-2 text-sm rounded-md`

5. **Form elements (if any inputs in onboarding):**
   - Add `shadow-sm` and `focus:ring-2 focus:ring-blue-500/40`

6. **Illustrations/icons (if any):**
   - Ensure consistent sizing and spacing
   - If SVG icons, ensure `text-blue-500 dark:text-blue-400` for branded feel

7. **Transitions between steps:**
   - Ensure smooth step transitions using Svelte transitions (fly, fade, or slide)
   - If no transition exists, add `fly={{ x: 20, duration: 200 }}` for subtle slide effect between steps

8. **Skip button:**
   - Should be clearly secondary: `text-sm text-gray-500 dark:text-gray-400 hover:text-gray-700 dark:hover:text-gray-300`
   - Not blue (blue = forward action)
  </action>
