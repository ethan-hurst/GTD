<task type="auto">
  <name>Task 1: Establish design tokens and global polish utilities in app.css</name>
  <files>src/app.css</files>
  <action>
Extend src/app.css with Tailwind 4's @theme directive to define design tokens. Keep the existing @import "tailwindcss" and @custom-variant dark lines. Add AFTER them:

1. **@theme tokens** -- Define semantic spacing, typography, and color tokens:
   - Spacing scale references: document that standard padding is p-4 (content), p-6 (sections), p-8 (page-level)
   - No custom spacing values needed -- use Tailwind's built-in scale consistently

2. **Global base styles** using @layer base:
   - Set `body` to use `font-family: system-ui` (already default, but be explicit)
   - Set default `text-rendering: optimizeLegibility` and `-webkit-font-smoothing: antialiased` for professional text
   - Set selection color: `::selection { background: theme(--color-blue-100); }` and dark variant

3. **Utility classes for consistent patterns** using @layer utilities:
   - `.focus-ring` -- Shorthand for `focus:ring-2 focus:ring-blue-500/40 focus:ring-offset-2 focus:ring-offset-white dark:focus:ring-offset-gray-950 focus:outline-none` (reusable across all interactive elements)
   - `.btn-base` -- Base button styles: `font-medium rounded-md transition-all duration-150 ease-out` with the focus-ring pattern baked in
   - `.shadow-soft` -- A refined shadow: `0 1px 3px 0 rgb(0 0 0 / 0.06), 0 1px 2px -1px rgb(0 0 0 / 0.06)` for cards
   - `.shadow-elevated` -- A stronger shadow for modals/dropdowns: `0 4px 6px -1px rgb(0 0 0 / 0.07), 0 2px 4px -2px rgb(0 0 0 / 0.07)`

4. **Smooth scrollbar styling** for webkit:
   - Thin, subtle scrollbars that don't dominate the UI
   - `::-webkit-scrollbar { width: 6px; }` with gray-200/gray-800 track colors

5. **Transition timing tokens** -- Define a CSS custom property `--transition-fast: 150ms` and `--transition-normal: 200ms` in @theme

Do NOT:
- Remove or modify existing lines (keep @import and @custom-variant)
- Add component-level classes (no .card, .modal -- those stay in components)
- Add color values that don't already exist in Tailwind's palette
- Create a full design system -- just tokens and a few critical utilities
  </action>
  <verify>
Run `npx vite build` -- should compile without errors. Check that app.css has @theme directive with tokens, @layer base with body styles, and @layer utilities with .focus-ring and .btn-base classes.
  </verify>
  <done>app.css contains design tokens in @theme, base layer with antialiased text and selection colors, utility layer with focus-ring and btn-base classes, and scrollbar styling. Build passes cleanly.</done>
</task>
