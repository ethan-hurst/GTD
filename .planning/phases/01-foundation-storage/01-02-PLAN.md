         4. Show success feedback (brief "Exported!" text or checkmark)
         5. Clear loading state
       - Style button with professional look: bg-blue-600 hover:bg-blue-700 text-white rounded px-4 py-2

       **Import section:**
       - Section header: "Data Import"
       - Description text: "Restore from a backup file. This will replace ALL current data."
       - Warning text in amber/yellow: "Warning: Importing will overwrite all existing data."
       - Hidden file input (`<input type="file" accept=".json">`)
       - "Import Data" button that triggers file input click
       - On file selected:
         1. Read file with FileReader
         2. Set loading state
         3. Call `await importDatabase(fileContent)`
         4. Show success feedback
         5. Clear loading state
       - Error handling: try/catch around import, show error message if validation fails or JSON is malformed
       - Use `$state` for loading and status message states

       **Layout:**
       - Card-style sections with border and padding
       - Spacing between sections
       - Max-width container (e.g., max-w-2xl) for readable line lengths

    2. Create `src/service-worker.js` (NOT .ts -- SvelteKit service workers use plain JS):
       - Import `{ build, files, version }` from `'$service-worker'`
       - Define cache name: `cache-${version}`
       - Define assets: `[...build, ...files]`

       **Install event:**
       - Open cache, add all assets with `cache.addAll(ASSETS)`
       - Call `self.skipWaiting()` to activate immediately

       **Activate event:**
       - Delete old caches (any cache key !== current CACHE name)
       - Call `self.clients.claim()` to take control immediately

       **Fetch event:**
       - Only handle GET requests (return early for others)
       - For known assets (url.pathname in ASSETS): serve from cache first, fall back to network
       - For other GET requests: try network first, cache successful responses (status 200), fall back to cache if network fails
       - This gives: instant load for app assets, fresh data for API calls (though we have none in Phase 1)

       Follow the exact pattern from RESEARCH.md Pattern 3, but add `self.skipWaiting()` in install and `self.clients.claim()` in activate for immediate activation.
  </action>
