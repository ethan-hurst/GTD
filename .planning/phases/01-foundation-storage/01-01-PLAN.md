  <name>Task 2: Create Dexie.js database schema, CRUD operations, and export/import modules</name>
  <files>

  <action>
    Create the complete data layer using Dexie.js 4.x patterns from RESEARCH.md:

    1. Create `src/lib/db/schema.ts`:
       - Define `GTDItem` interface with fields: `id: number`, `title: string`, `type: 'inbox' | 'next-action' | 'project' | 'waiting' | 'someday'`, `notes: string`, `created: Date`, `modified: Date`
       - Define `GTDList` interface with fields: `id: number`, `name: string`, `type: string`
       - Create Dexie database instance named "GTDDatabase" using `EntityTable` pattern for type safety:
         ```typescript
         export const db = new Dexie("GTDDatabase") as Dexie & {
           items: EntityTable<GTDItem, "id">;
           lists: EntityTable<GTDList, "id">;
         };
         ```
       - Define version 1 schema: `items: "++id, type, created, modified"`, `lists: "++id, name, type"`
       - Export types and db instance

    2. Create `src/lib/db/operations.ts`:
       - Import `db` and types from schema
       - `addItem(item: Omit<GTDItem, 'id' | 'created' | 'modified'>): Promise<number>` -- adds with auto-generated timestamps
       - `updateItem(id: number, changes: Partial<GTDItem>): Promise<number>` -- updates modified timestamp
       - `deleteItem(id: number): Promise<void>`
       - `getItem(id: number): Promise<GTDItem | undefined>`
       - `getAllInbox(): Promise<GTDItem[]>` -- filters by type='inbox', sorted by created date
       - `getItemsByType(type: GTDItem['type']): Promise<GTDItem[]>`
       - All operations use `await db.items.*` (NOT within manual transactions for simple ops)

    3. Create `src/lib/db/export.ts`:
       - `exportDatabase(): Promise<string>` -- iterates `db.tables`, calls `toArray()` on each, returns JSON.stringify with metadata (version, exported timestamp, data object keyed by table name). Use 2-space indentation for readability.
       - `importDatabase(jsonData: string): Promise<void>` -- parses JSON, validates structure (must have `data` object with known table names), wraps in `db.transaction('rw', db.tables, ...)` to clear existing data then `bulkPut` imported data. Throw descriptive error if validation fails.
       - `downloadJSON(data: string, filename: string): void` -- creates Blob, generates object URL, triggers download via anchor element click, revokes URL
       - Validation in importDatabase: check that parsed JSON has `data` property, check that each table name in data exists in db.tables, wrap entire operation in try/catch with meaningful error messages

    ANTI-PATTERNS TO AVOID (from RESEARCH.md):
    - Never mix fetch/setTimeout in Dexie transactions
    - Never catch errors without re-throwing in transactions (breaks abort)
    - Use Omit<> for add operations to exclude auto-generated fields
  </action>

  <action>
    Create state management using Svelte 5 $state runes (NOT writable stores):

    1. Create `src/lib/stores/storage.svelte.ts`:
       - Export `StorageStatus` class with $state rune fields:
         - `isPersistent = $state(false)` -- whether persistent storage is granted
         - `quota = $state(0)` -- total storage quota in bytes
         - `usage = $state(0)` -- current storage usage in bytes
         - `lastSaveTime = $state<Date | null>(null)` -- last successful database write time
         - `lastCheck = $state<Date | null>(null)` -- last time quota was checked
       - Methods:
         - `async checkPersistence()` -- reads `navigator.storage.persisted()`, updates isPersistent
         - `async requestPersistence(): Promise<boolean>` -- calls `navigator.storage.persist()`, updates isPersistent, returns result
         - `async updateQuota()` -- calls `navigator.storage.estimate()`, updates quota/usage/lastCheck
         - `recordSave()` -- sets lastSaveTime to new Date() (called after successful db writes)
       - Guard all navigator.storage calls with feature detection (`if (navigator.storage && navigator.storage.persist)`)
       - Export singleton: `export const storageStatus = new StorageStatus();`

    2. Create `src/lib/stores/theme.svelte.ts`:
       - Export `ThemeStore` class with $state rune field:
         - `current = $state<'light' | 'dark' | 'system'>('system')`
       - Constructor: read from `localStorage.getItem('theme')` if in browser, call `apply()`
         - Guard with `typeof window !== 'undefined'` check
       - Methods:
         - `apply()` -- determine if dark mode should be active (current === 'dark' OR current === 'system' AND prefers-color-scheme is dark), toggle `dark` class on `document.documentElement`
         - `set(theme: 'light' | 'dark' | 'system')` -- update current, persist to localStorage (or remove if 'system'), call apply()
         - `listen()` -- add matchMedia listener for system preference changes, return cleanup function. Call this in onMount.
       - Export singleton: `export const theme = new ThemeStore();`

    IMPORTANT: These are `.svelte.ts` files (not `.ts`) because they use $state runes, which require the Svelte compiler. The `.svelte.ts` extension tells SvelteKit to process them through the Svelte compiler.
  </action>
