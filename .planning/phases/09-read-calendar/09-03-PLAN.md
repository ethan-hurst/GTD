<feature>
  <name>Outlook Calendar Sync Service</name>
  <files>src/lib/services/graph/calendar.ts, src/lib/services/graph/calendar.test.ts</files>
  <behavior>
    The calendar sync service has three main functions:

    1. **mapOutlookEvent(outlookEvent, calendarId) -> Partial&lt;CalendarEvent&gt;** (pure function, highly testable)
       - Maps Microsoft Graph event fields to CalendarEvent fields:
         - subject -> title
         - start.dateTime + start.timeZone -> startTime (parsed to Date)
         - end.dateTime + end.timeZone -> endTime (parsed to Date)
         - isAllDay -> allDay
         - location.displayName -> location
         - bodyPreview -> notes (truncated to 500 chars)
         - id -> outlookId
         - @odata.etag -> outlookETag
         - calendarId param -> outlookCalendarId
         - syncSource = 'outlook'
         - lastSyncedAt = new Date()
       - Special cases:
         - All-day events: Use date string directly (YYYY-MM-DD), create start/end as local midnight. Do NOT apply timezone conversion.
         - Non-all-day events: Parse dateTime with timeZone to create UTC-aware Date objects.
         - Missing location: set to undefined (not empty string)
         - Missing body: set notes to undefined

       Test cases:
         - mapOutlookEvent({subject: "Team Standup", start: {dateTime: "2026-02-03T09:00:00", timeZone: "America/New_York"}, end: {dateTime: "2026-02-03T09:30:00", timeZone: "America/New_York"}, ...}) -> { title: "Team Standup", startTime: Date, endTime: Date, allDay: false, syncSource: 'outlook' }
         - All-day event: isAllDay=true, start.dateTime="2026-02-03", end.dateTime="2026-02-04" -> allDay: true, startTime is local midnight Feb 3, endTime is local midnight Feb 4
         - Event with location -> location set
         - Event without location -> location undefined
         - Event with @removed annotation -> returns null (deletion marker)

    2. **fetchCalendars() -> OutlookCalendar[]**
       - Calls GET /me/calendars
       - Returns array of { id, name, color, isDefaultCalendar }

    3. **syncCalendarEvents(calendarId, existingDeltaLink?) -> SyncResult**
       - If existingDeltaLink provided: use it for delta query
       - If no deltaLink: perform initial sync with date range (6 months back, 12 months forward)
       - Process response:
         - For each event WITHOUT @removed: upsert into Dexie (match on outlookId, update if ETag differs, insert if new)
         - For each event WITH @removed: soft-delete from Dexie (set deleted=true)
         - Return new deltaLink for storage
       - Handle syncStateNotFound error: clear deltaLink, retry with full sync
       - Use bulk operations (build lookup map of existing outlookIds, then bulk write) to avoid N+1 queries

       SyncResult type:
       ```typescript
       interface SyncResult {
         success: boolean;
         deltaLink?: string;
         eventsAdded: number;
         eventsUpdated: number;
         eventsDeleted: number;
         error?: string;
       }
       ```
  </behavior>
  <implementation>
    Create src/lib/services/graph/calendar.ts with:

    1. OutlookEvent type (Microsoft Graph event shape — not the full type, just the fields we use)
    2. mapOutlookEvent as an exported pure function (for testability)
    3. fetchCalendars using graphFetch
    4. syncCalendarEvents using graphFetchAll for pagination + Dexie bulk operations

    For syncCalendarEvents, the implementation should:
    - Build initial URL: `/me/calendars/${calendarId}/calendarView/delta?startDateTime=...&endDateTime=...` for initial sync
    - Or use existingDeltaLink directly for incremental sync
    - Call graphFetchAll to collect all pages
    - Separate events into upserts and deletions (check @removed)
    - Load existing events by outlookId from Dexie (single query: db.events.where('outlookId').anyOf(ids))
    - Build lookup map for efficient matching
    - Bulk upsert: db.events.bulkPut for new/updated, db.events.update for deletions (soft-delete)
    - Store new deltaLink in syncMeta table

    For tests, mock graphFetch/graphFetchAll at module level. Test mapOutlookEvent as a pure function with real inputs/outputs. Test syncCalendarEvents with mocked API responses and mocked Dexie.
  </implementation>
