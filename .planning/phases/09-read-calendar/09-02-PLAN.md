Create `src/lib/services/graph/client.ts` as a thin wrapper around fetch for Microsoft Graph API calls.

```typescript
import { acquireToken } from './auth';

const GRAPH_BASE_URL = 'https://graph.microsoft.com/v1.0';
const MAX_RETRIES = 3;

interface GraphResponse<T = any> {
  ok: boolean;
  status: number;
  data?: T;
  error?: string;
  nextLink?: string;   // @odata.nextLink for pagination
  deltaLink?: string;  // @odata.deltaLink for delta queries
}
```

Export a `graphFetch` function:
```typescript
export async function graphFetch<T = any>(
  path: string,
  options: RequestInit = {}
): Promise<GraphResponse<T>>
```

Implementation requirements:
1. Acquire Bearer token via `acquireToken()` before each request
2. Set headers: `Authorization: Bearer ${token}`, `Content-Type: application/json`
3. Prepend `GRAPH_BASE_URL` to path unless path is a full URL (for following nextLink/deltaLink URLs)
4. Parse JSON response
5. Extract `@odata.nextLink` and `@odata.deltaLink` from response and include in GraphResponse
6. Handle HTTP 429 (Too Many Requests): read `Retry-After` header (seconds), wait that duration, then retry. Max 3 retries.
7. Handle HTTP 401 (Unauthorized): attempt token refresh and retry once
8. Handle HTTP 412 (Precondition Failed): return as error (ETag conflict — caller decides)
9. On network errors, return `{ ok: false, status: 0, error: message }`

Also export a convenience function for paginated requests:
```typescript
export async function graphFetchAll<T = any>(
  path: string
): Promise<{ ok: boolean; data: T[]; deltaLink?: string; error?: string }>
```

This follows `@odata.nextLink` until it gets `@odata.deltaLink` or no more pages, collecting all items from the `value` array across pages.

Important: When path starts with 'https://', use it as-is (don't prepend GRAPH_BASE_URL). This is needed for delta link and next link URLs which are full URLs returned by Graph API.

Create `src/lib/stores/auth.svelte.ts` following the Svelte 5 $state runes pattern used by existing stores (e.g., sync.svelte.ts):

```typescript
class AuthStore {
  isAuthenticated = $state(false);
  isLoading = $state(false);
  userName = $state<string | null>(null);
  userEmail = $state<string | null>(null);
  error = $state<string | null>(null);

  async init(): Promise<void> { ... }
  async connect(): Promise<void> { ... }
  async disconnect(): Promise<void> { ... }
}

export const authState = new AuthStore();
```

init() should:
1. Guard with `if (typeof window === 'undefined') return` for SSR safety
2. Call `getMsalInstance()` to ensure redirect promise is handled
3. Call `getAuthenticatedAccount()` to check if already logged in
4. If account exists, set isAuthenticated=true, userName=account.name, userEmail=account.username
5. Catch errors and set error state

connect() should:
1. Set isLoading=true
2. Call `loginWithMsal()` (this triggers redirect — page will reload)
3. On error, set error state, isLoading=false

disconnect() should:
1. Set isLoading=true
2. Clean up Outlook events from Dexie: delete all CalendarEvents where syncSource === 'outlook'
3. Clear all syncMeta records
4. Call `logoutFromMsal()` (triggers redirect)
5. Reset all state

Create `src/lib/components/OutlookAuthButton.svelte`:

- **Not connected**: Show "Connect to Outlook" button with Microsoft icon. Blue background, white text. On click, call `authState.connect()`.
- **Connected**: Show user name and email, plus a "Disconnect" button. Gray/red disconnect button. On click, call `authState.disconnect()`.

Show loading spinner while isLoading is true.
Show error message if error is set (red text below button).

Style with Tailwind classes matching existing app design patterns:
- Use rounded, shadow-sm, hover states consistent with other buttons
- Support dark mode
- Minimum touch target 44px (min-h-10 pattern)

Important: Use `$effect` to call `authState.init()` on mount (in the component or wherever this is first rendered). Do NOT call init in the store constructor (SSR safety).
