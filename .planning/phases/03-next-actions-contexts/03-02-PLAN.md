**Create `src/lib/components/ActionList.svelte`:**

Main list component for next actions. Supports drag-and-drop reordering via svelte-dnd-action, renders ActionItem for each action, and handles completion with undo toast.

Structure:
- Header bar: Shows current view name ("All Actions" or selected context names), action count, bulk action buttons when items selected
- Action list: Rendered inside a `use:dndzone` container with flip animations
- Empty state: Friendly message when no actions exist in current context ("No actions in @computer" or "No next actions yet. Process your inbox to create actions.")

**Drag-and-drop implementation** (per RESEARCH.md Pattern 2):
```svelte
<script lang="ts">
  import { dndzone } from 'svelte-dnd-action';
  import { flip } from 'svelte/animate';
  import { actionState } from '$lib/stores/actions.svelte';
  import { completeAction, reorderActions, bulkCompleteActions } from '$lib/db/operations';
  import { toast } from 'svelte-5-french-toast';

  let dragItems = $state(actionState.items);

  // Keep dragItems in sync with actionState
  $effect(() => {
    dragItems = [...actionState.items];
  });

  function handleDndConsider(e: CustomEvent) {
    dragItems = e.detail.items;
  }

  async function handleDndFinalize(e: CustomEvent) {
    dragItems = e.detail.items;
    await reorderActions(dragItems.map(item => item.id));
    await actionState.loadActions();
  }
</script>
```

**Completion handling:**
- `handleComplete(id)`: Call `completeAction(id)`, show toast with undo button. After 1s delay, reload actions list. If undo clicked, call the returned undo function and reload.
- Toast message: `"Task title" completed` with "Undo" action. Duration: 5000ms.
- Avoid RESEARCH.md Pitfall 1 (race condition): Use the local isCompleting state in ActionItem to manage visual feedback independently from list reload.

**Batch completion:**
- When items are selected (actionState.selectedIds.length > 0), show "Complete selected" button in header
- On click: Call `bulkCompleteActions(selectedIds)`, show single toast "N actions completed - Undo", clear selection, reload.
- Per RESEARCH.md Pitfall 5: Single toast for batch, not individual toasts.

**"All" view grouping:**
- When actionState.isAllView is true, group actions by context under headings
- Group header: "@computer", "@phone", etc. with count
- Actions without context: Show under "No context" heading
- Each group is independently drag-reorderable (within its context)

**Empty state:**
- If no actions at all: "No next actions yet. Process your inbox to create actions." with a link to "/" (inbox).
- If filtered by context with no results: "No actions for [context names]. Switch contexts or process your inbox."
