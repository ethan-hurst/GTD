<task type="auto">
  <name>Task 2: Action and Context database operations</name>
  <files>src/lib/db/operations.ts</files>
  <action>
Add the following functions to `src/lib/db/operations.ts` (keep all existing functions):

**Context operations:**
- `getAllContexts(): Promise<Context[]>` - Returns all contexts sorted by sortOrder.
- `addContext(name: string): Promise<number>` - Creates a new context. Name must start with "@". Set sortOrder = max(existing sortOrder) + 1. Returns the new context ID.
- `updateContext(id: number, changes: Partial<Context>): Promise<number>` - Update context fields (name, color, icon, sortOrder).
- `deleteContext(id: number): Promise<void>` - Delete a context. Also clear the `context` field on any items that reference this context name.

**Action operations:**
- `getActionsByContext(contexts: string[]): Promise<GTDItem[]>` - Query items where type='next-action' AND completedAt is undefined/null AND context is in the provided array. Sort by sortOrder (ascending, nulls last), then by created date (ascending, FIFO). Use `.where('type').equals('next-action').filter(...)` pattern since OR context queries need in-memory filter (see RESEARCH.md Pitfall 3).
- `getAllNextActions(): Promise<GTDItem[]>` - Query ALL next-actions (not completed). Useful for "All" view. Sort by sortOrder then created.
- `completeAction(id: number): Promise<() => Promise<void>>` - Set completedAt = new Date() and modified = new Date(). Return an undo function that clears completedAt and sets modified = new Date(). This pattern supports the toast undo flow.
- `undoCompleteAction(id: number): Promise<void>` - Clear completedAt, set modified = new Date(). (Also callable directly if needed.)
- `reorderActions(orderedIds: number[]): Promise<void>` - Takes an array of item IDs in their new order. Sets sortOrder = index for each. Batch update using Promise.all. Sets modified = new Date() on each.
- `bulkCompleteActions(ids: number[]): Promise<() => Promise<void>>` - Batch complete multiple actions. Returns undo function that reverts all.

Import the `Context` type from schema.ts. Use `db.contexts` for context operations.

```typescript
export class ActionState {
  // State
  items = $state<GTDItem[]>([]);
  contexts = $state<Context[]>([]);
  selectedContexts = $state<string[]>([]);  // Empty = "All" view
  selectedIds = $state<number[]>([]);        // For batch operations
  expandedId = $state<number | null>(null);  // For detail panel

  // Derived
  itemCount = $derived(this.items.length);
  isAllView = $derived(this.selectedContexts.length === 0);

  // Load actions based on current context filter
  async loadActions() { ... }

  // Load all contexts from database
  async loadContexts() { ... }

  // Toggle a context filter on/off (multi-select)
  toggleContext(contextName: string) { ... }

  // Select a single context (replaces selection)
  selectContext(contextName: string) { ... }

  // Show all actions (clear context filter)
  showAll() { ... }

  // Selection management (same pattern as inbox)
  toggleSelection(id: number) { ... }
  clearSelection() { ... }
  selectAll() { ... }

  // Expand/collapse detail panel
  expandItem(id: number) { ... }
}

export const actionState = new ActionState();
```

Implementation details:
- `loadActions()`: If `selectedContexts` is empty (All view), call `getAllNextActions()`. Otherwise call `getActionsByContext(selectedContexts)`.
- `loadContexts()`: Call `getAllContexts()` and set `this.contexts`.
- `toggleContext()`: Add/remove from selectedContexts array, then call loadActions(). If removing the last selected context, switch to All view.
- `selectContext()`: Set selectedContexts to [contextName], then call loadActions().
- `showAll()`: Set selectedContexts to [], then call loadActions().
- Selection methods: Mirror the inbox.svelte.ts pattern exactly.
- `expandItem()`: Toggle pattern same as inbox.

Import from operations.ts: getActionsByContext, getAllNextActions, getAllContexts.
Import from schema.ts: GTDItem, Context types.
